# Input 输入指南入口

> Load when: 已选定方法论、要把某个提示词槽位真正填好，需要某个字段的具体填法或参考模板——景别/构图/运镜各选项的语言效果与选用判据、整片风格怎么定、参考图怎么投、时间光影/地点氛围 token。
> Avoid: 只是要选哪种方法论（去 `../../video-gen-pro/references/methods.md`）、只是要厘清剧本/资产/方法论边界（去 `../../video-gen-pro/references/concepts.md`）、或只要某类资产图的绘图提示词（去 `../../video-gen-assets/references/` 对应 sheet）。
> Pairs with: `input/README.md` 是输入指南二级索引；`../../video-gen-director/references/director-method.md` / `../../video-gen-storyboard/references/storyboard-board.md` 定义有哪些槽位，本维度只管这些槽位「填什么、怎么填」。

本文件是**输入指南（Input Guide）**的运行时入口，只负责路由。输入指南是一个**横切所有方法论**的填料层：方法论（导演法/故事板法）定义提示词的**骨架结构**和有哪些槽位，输入指南是这些槽位「填什么、怎么填」的**参考知识库**，被所有方法论**共同调用**。它不与剧本/资产/方法论并列同类——剧本回答「拍什么」、资产回答「用什么料」、方法论回答「用什么骨架」，输入指南回答「骨架的每个槽怎么填好」。横切维度的概念边界以 `../../video-gen-pro/references/concepts.md` 为准。

不要一次性加载全部内容；先判断要填哪个槽位，再读对应子文件。

## Input Route

| Need | Read | 填哪个槽位 |
| ---- | ---- | ---------- |
| 景别（含焦点/景深）各选项的画面语言效果、选用判据、prompt 写法 | `input/shot-scale.md` | 导演法【画面内容】的景别；故事板法每格的取景 |
| 构图（含机位角度）各选项的画面语言效果、选用判据、prompt 写法 | `input/composition.md` | 导演法【画面内容】的构图；故事板法每格的构图 |
| 运镜手法各选项的画面语言效果、选用判据、prompt 写法，含五类镜头词库速查 | `input/camera-move.md` | 导演法【画面内容】的运镜手法；故事板法每格的运动目标 |
| 整片风格核心（画风/题材流派）的可挑选模板、抽象词→可见元素的翻译 | `input/style-core.md` | 导演法【氛围与画质】的风格核心；设定流的视觉圣经；故事板法整片风格 |
| 整片视觉基调（器材/画幅/胶片质感）的可挑选模板 | `input/visual-tone.md` | 导演法【氛围与画质】的视觉基调 |
| 随段落微调的色彩与影调（冷暖/对比/明暗/调色/颗粒）的可挑选模板 | `input/lighting-tone.md` | 导演法【氛围与画质】的色彩与影调；故事板法每段调色 |
| 时间光影 / 地点氛围的中英文锚点速查 | `input/scene-tokens.md` | 场景图的时间/氛围字段 |
| 参考图怎么投（按需投放、图内零文字）、参考素材怎么编号 | `input/image-input.md` | 任何方法论最后往视频模型喂参考图的 payload |

## 横切关系（为什么独立成维）

- **跨方法论共用**：景别/构图/运镜——导演法在【画面内容】填，故事板法画母图也要定每格景别构图；风格库——两法都要定整片风格；图片指南——两法最后都往 Seedance 喂参考图。挂在任一单方法下，另一方够不着或要重复，所以独立成横切维度。
- **有独立实体正文**（风格库、填法理论、token 库、图片指南），不是「指向别处的指针」——它是被多方共用、且有独立实体内容的填法库。
- **不破坏三分基石**：输入指南不是第四个并列概念，是叠在剧本/资产/方法论之上的另一根轴；concepts.md 的三分表保持不变，横切维度单独成节说明它「横切」三者（尤其方法论）。
