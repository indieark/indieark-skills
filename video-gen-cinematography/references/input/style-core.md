# Style Core 画风 / 风格核心填法

> Load when: 要定整片画风——填导演法【氛围与画质】的风格核心（整片不变的最底层基调，是什么质感的世界、哪个题材流派），或设定流的视觉圣经，需要可挑选的风格模板和「抽象词→可见元素」的翻译。
> Avoid: 要定用什么器材/画幅拍的（去 `visual-tone.md`）、随段落微调的色彩与影调（去 `lighting-tone.md`）、单镜时间/氛围 token（去 `scene-tokens.md`）、或景别构图运镜（去 `shot-scale.md` / `composition.md` / `camera-move.md`）。
> Pairs with: `../../../video-gen-director/references/director-method.md`【氛围与画质】定义有哪些子字段，本文件管整片不变的风格核心填什么；视觉基调拆在 `visual-tone.md`、随段落微调的色彩与影调拆在 `lighting-tone.md`；`../../../video-gen-assets/references/setting-director.md` 的视觉圣经/Style Bible 是设定流里的一步，本库是它的展开模板池。

风格核心是横切两种方法论的填法知识：导演法在【氛围与画质】填，故事板法定整片风格，设定流的视觉圣经引用。它定「这是个什么质感的世界」——写实、2D 还是 3D，哪个题材流派，画面像什么材质，是整片几乎不变的最底层基调。本文件提供可挑选的风格模板库（即梦官方 94 风格），并给出把抽象词翻译成可见元素的规则。器材/画幅层面的视觉基调拆在 `visual-tone.md`，随段落微调的色彩与影调拆在 `lighting-tone.md`。

## Style Bible 翻译规则

风格规则必须把抽象词翻译成可见元素，不能只写“高级感/电影感”这类抽象词：

```text
风格：<题材/平台>，<画幅>，<主色调>，<光线方向>，<材质>，
<镜头速度>，<景深/颗粒/质感>，<声音或节奏倾向>。
```

示例：

```text
9:16 竖屏短剧，暗童话悬疑，低饱和红与冷绿，门内暖光和门外冷光对比，
木门粗糙纹理，浅景深，缓慢推近，轻微胶片颗粒，安静到危险逼近。
```

这套「抽象词 → 可见元素」翻译对应导演法【氛围与画质】的两个**整片不变**子字段：风格核心、视觉基调。第三层「色彩与影调」随段落微调，模板池拆在 `lighting-tone.md`；「氛围基调」随剧本变，不固定进风格模板。

## 风格核心模板池（即梦官方 94 风格）

风格核心定「这是个什么质感的世界」——写实、2D 还是 3D，哪个题材流派，画面像什么材质。它是整片几乎不变的最底层基调，定了之后色彩与影调只在它内部微调。

风格名直接是 prompt 锚点：Seedance 吃中文，把官方中文风格名整词写进【氛围与画质】的「风格核心」即可触发，再用「可见元素」补足细节、「prompt 写法」给成套句式。下列**分类与排序逐字对齐即梦风格库的官方三个 tab——真人 / 2D / 3D**，每个风格配官方渲染封面（仅本地参考，不入库）。注意几个反直觉的官方归类：橙黄/荒诞高调白/蓝橙/紫色四个「色调电影」官方归**真人**（是统一调色的实拍风格，不是独立色调层）；中国神话官方归 **2D**（插画神话，非真人史诗）；复古 Y2K 奇幻官方归 **3D**（立体渲染奇幻，非实拍）。
