# Image Input 图片使用指南

> Load when: 要把参考图喂给视频模型，需要决定喂哪几张、喂多细、图内能不能带文字，以及参考素材在 prompt 里怎么编号。
> Avoid: 只需要某类资产图怎么**生成**（去 `../../../video-gen-assets/references/` 对应 sheet）、或资产**做**多重（梯度 L0→L2，见 `../../../video-gen-pro/references/concepts.md` 与各 sheet）。
> Pairs with: `../../../video-gen-assets/references/principles.md` 管资产链路理念；本文件管喂图侧的纯净度、按需投放和编号；`../../../video-gen-assets/references/seedance-handoff.md` 把图片资产转成 Seedance payload。

参考图按用途分两类，这里管三件事：图内文字策略（Reference Purity）、整次输入喂多少图喂多细（Input Economy）、参考素材在 prompt 里怎么编号（全能参考写法）。这三件都是「往视频模型喂参考图」时横切所有方法论的通用规则，不属于任一单方法。

## Reference Purity

视觉资产按用途分两类，图内文字策略相反：

- 给人、规划或 Agent 复盘看的中间产物（故事板母图、脚本拼版信息图）可以、甚至应该带文字：CUT 表、时间码、对白、栏目标题。
- 直接作为视频模型参考输入的资产（角色卡、场景卡）必须纯图像、零语义文字。图内的标题、字段表、栏目名和文字标签会被视频模型当成要还原的画面元素，加重心智负担并污染成片。

判据是“这张图给谁当输入”。角色卡和场景卡的设定信息（角色名、脸部 DNA、空间 DNA、光线方向、道具位置）写进提示词正文和资产登记，不画进图里；纯几何测量线和对齐参考线不算文字，可以保留。

## Input Economy

Reference Purity 管“单张图里别混文字”；这一节管“整次生成别堆太多图、每张别过细”。两条轴不同：图档梯度（L0→L2，见各 asset sheet）按**复用强度**决定一个资产**做**多重；这里按**当前镜头实际需要**决定**喂**模型哪几张、喂多细。

视频模型一次能稳定消化的参考图是有限的：**参考图数量 × 单图细节量**整体超标，模型反而读不懂——它要么自行判断主次把资产混在一起，要么被次要资产的细节带偏，稀释真正的主体。所以喂图的原则是**按当前镜头精选最小够用集**，不是把项目里所有 master、所有视图、所有特写一股脑全塞进去。

落地判断：

- **按角色挑档喂，不一律拉满**：当前镜头的主体角色给它的 master（L2 多视图或 L1 三视图）；只是背景里一闪而过、只要个剪影或大致轮廓的角色和物体，给一张简单单图（L0）就够，不必为它上三视图或多视图。
- **L2 master 的捆绑特写按镜头挑一张，不是每镜全喂**：主角有一组面部/表情特写时，当前镜头要哪个情绪就喂那一张，不要把整组特写都塞进同一次输入。
- **场景按复用喂**：本镜不复用的场景给单图即可，不必上多视角联合图或正交四视图。
- **每张都得有用途**：进入这次输入的每张参考图，都要能在 prompt 里说清它管什么（主体/空间/风格/某个细节）；说不清用途的图，就是该被精简掉的过载。

判据是“这个镜头到底需要锁住什么”，而不是“这个项目一共做了多少资产”。资产做得多是复用储备，喂得多是心智负担——两者分开。

这一节是在「相关性筛选」之上再加的一层。`../../../video-gen-script/references/generation-workflow.md`（只提交当前段需要的输入、无关角色卡/场景卡/音频不进 payload）和 `../../../video-gen-script/references/multi-segment-continuity.md`、`../../../video-gen-storyboard/references/script-composed-storyboard.md`（按段控制每轮输入负载、本段出现什么才提交什么）先用**相关性**筛掉无关输入；Input Economy 接着在**留下的相关输入里**按**总量与细节量**精选最小够用集。两步递进，不是重复：先筛相关，再控过载。

## 全能参考写法

参考素材要在 prompt 中显式编号。编号按 payload 中同类素材的输入顺序计算：第 1 个 `reference_image` 是 `参考图1` / `图片1`，第 2 个是 `参考图2` / `图片2`；第 1 个 `reference_video` 是 `参考视频1` / `视频1`；第 1 个 `reference_audio` 是 `参考音频1` / `音频1`。这和火山官方示例一致，比笼统写“参考所有素材”稳定。

```text
参考图1用于主角面部和服装，不改变发型和主色。
参考图2用于场景氛围和灯光色彩。
参考视频1用于动作节奏和镜头跟拍方式，不复制具体背景。
参考音频1用于鼓点节奏和紧张情绪，画面动作跟随鼓点加速。
```

编号写法和 Input Economy 配合：先按 Input Economy 精选最小够用集，再给留下的每张图编号并写清用途——能编号、能说清用途的图才该进 payload，说不清用途的就是该精简掉的过载。
