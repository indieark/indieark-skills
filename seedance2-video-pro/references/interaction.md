# Interactive Intent Reference

> Load when: 需要判断是否追问、给方向选项，或直接执行。
> Avoid: 用户只问 API 语法、CLI 参数、任务状态查询，不需要导演问询。
> Pairs with: `router.md` 决定后续任务类型；`director.md` 接收澄清结果写 prompt。

Seedance 2.0 生成有成本，且用户经常只给半成品意图。Skill 必须先判断是否需要问询用户意见，再决定生成、给方向、还是继续澄清。

## Interaction Gate

每次请求先过问询门：

```text
用户请求
-> 信息是否足够
-> 是否有多种合理方向
-> 是否存在素材角色不清或成本风险
-> ask / propose options / proceed
```

## Ask / Propose / Proceed

| State | Action |
|-------|--------|
| 缺主体、用途、画幅、时长、声音、素材角色中的关键项 | Ask |
| 用户给了明确场景但缺少低风险参数 | Ask 1-3 个关键问题，或用默认值并说明 |
| 有 2-3 个创意方向都合理 | Propose options |
| 素材多但用途不明 | Ask material role |
| 用户明确给出目标、场景、素材角色和参数 | Proceed |
| 用户要求“直接生成/别问太多”且风险低 | Proceed with conservative defaults |

## Question Design

一次最多问 3 个问题。问题要能直接影响生成结果，避免问泛泛偏好。

优先顺序：

1. 目标和用途：广告、短剧、产品展示、角色动作、氛围片、社媒短视频、编辑/延长。
2. 素材角色：首帧、尾帧、参考图、参考视频、参考音频。
3. 输出参数：画幅、时长、有声/无声。
4. 风格和限制：写实/动画/电影感、必须保留/禁止改变。
5. 质量取向：质量优先或速度/成本优先。

## Option Format

当用户需要选方向时，给 2-3 个短选项：

```text
我建议先选一个方向：
1. 克制氛围：单镜头慢移，重点是光线和空间。
2. 剧情转折：先安静展示，再出现一个细微变化。
3. 产品质感：更像高级展陈广告，突出材质和细节。
```

每个选项只说明差异，不写完整 prompt。用户选定后再写导演方案。

## Material Role Questions

素材不清时优先问：

- 这张图是首帧，还是只参考角色/产品/场景外观？
- 这段视频是要延长/编辑，还是只参考动作和运镜？
- 这段音频是参考节奏、音色、情绪，还是要生成近似声音？

## Stop Asking

满足以下条件就停止追问：

- 用户已明确核心目标和素材角色。
- 剩余缺口可用安全默认值。
- 用户要求直接执行。
- 继续追问只会拖慢生成，不会明显提高结果。

停止追问后，用一句话说明默认：

```text
我按 9:16、8 秒、有声、质量优先处理；参考图用于角色外观，参考视频用于动作，参考音频用于节奏。
```

## Do Not

- 不要一次问 5 个以上问题。
- 不要用户已经指定场景时还要求重新选场景。
- 不要把“问询”当作固定表单。
- 不要在用户只要 API 用法时强行进入导演问询。
- 不要为了追求完整而阻塞低风险任务。
