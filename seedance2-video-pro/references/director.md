# Director Reference

> Load when: 已完成交互/场景判断，需要写或裁剪 Seedance 导演 prompt。
> Avoid: 仍不清楚用户目标、素材角色或输入模式；先回到 `interaction.md` / `router.md`。
> Pairs with: `scenes.md` 决定保留哪些字段；`methods/README.md` 决定是否用场记板或分镜；`sound-design.md` 负责有声、无声、音效、对白和参考音频的可执行规则。

## 场景先行

导演链路不是固定填空模板。先判断用户要的场景，再选择要问什么、保留什么、删掉什么、怎么写 prompt。

场景路由细则见 `references/scenes.md`。当用户指定产品广告、短剧、角色动作、氛围片、社媒短视频、视频编辑/延长或多参考素材时，先按对应场景收束，不要把所有请求套成同一个“电影感分镜”公式。

导演方法分开存储在 `references/methods/`。场记板方法见 `references/methods/clapperboard.md`，分镜方法见 `references/methods/storyboard.md`。可以融合学习多种方法的判断思路，但一次任务只选择最适合的一种作为主结构，避免模板堆叠。需要人物设定图、场景设定图、故事板母图或脚本拼版故事板时，改读 `references/visual-assets.md` 写图像提示词、原子图清单或 composition manifest，再回到本文写 Seedance 视频 prompt。九宫格、四栏或制作设定板只是故事板母图的版式，不是导演方法本身。

## 写 Prompt 前置条件

复杂生成任务写最终视频 prompt 前，必须已经完成 `generation-workflow.md` 的导演流程：自然语言转剧本、剧本转镜头、镜头转角色/场景/道具资产依赖。没有故事板时，也不能省略这个步骤；只是把镜头清单直接转为下方详细分镜化视频 prompt。

如果任意非抽象镜头没有 `scene_ref`，先回到 `visual-assets.md` / `scene-sheet.md` 生成或登记场景卡。不要用一句“在某个场景中”替代场景资产，也不要把缺失的场景、道具和空间关系全塞进最终 prompt。

## 意图澄清模板

当用户只说“生成一个视频”时，优先问。用户已经给了明确场景时，只补真正影响结果的缺口：

1. 这是广告、故事、产品展示、角色动作，还是氛围片？
2. 用什么画幅和时长？如果不确定，短视频默认 `9:16`、6-8 秒。
3. 要有声还是无声？有声需要音乐、环境音、动作音效、对白，还是参考音频节奏？
4. 有参考图、参考视频、参考音频吗？每个素材要参考什么？

## Prompt 结构

推荐结构。按场景取用，不要求每次完整输出：

```text
成片目标：一句话说明视频要让观众记住什么。
参考关系：参考图1用于...，参考视频1用于...，参考音频1用于...
画面与风格：主体、场景、光线、色彩、质感。
镜头设计：景别、运镜、焦点、速度。
动作节奏：0-2秒...，2-5秒...，5-8秒...
声音设计：音乐、音效、对白；无声则写“无配乐无对白，仅画面动作”。
限制：保持主体一致，不改变关键物体，不加入无关文字或 logo。
```

## Prompt 详略策略

提示词详细程度取决于是否使用故事板母图。

### 使用故事板时

如果已经有 4K 故事板母图、角色卡、场景/背景卡和必要道具/产品卡，`video-prompt.txt` 不需要写得特别细。遵守“一图控制 + 文字解说图片”的策略：

- 让故事板母图承载镜头顺序、动作节奏、调度、灯光、情绪、声音区和必要文字说明。
- `video-prompt.txt` 只写参考图编号用途、读图顺序、运动目标、短声音 brief、字幕/标题文字策略和禁止项。
- 不要把故事板绘图提示词、镜头表和图内详细文字逐字复制进视频 prompt。
- 如果图里已有字幕、标题或画面文字，prompt 只说明“按故事板文字区显示 / 不新增其他文字 / 保持文字清晰不乱码”。
- **故事板适用条件**：清楚、可规划、需要一致性的镜头适合故事板；爆炸、粒子、烟雾、混乱战场或强抽象动态为主时，优先 prompt-heavy 或混合路径。
- **故事板确认边界**：即使用故事板，也必须在生成前独立确认 `generate_audio`、声音设计、说话人声、字幕/标题文字、画幅、清晰度/分辨率、时长、实际提交输入和最终 `video-prompt.txt`；故事板不能替代这些确认。
- **文字密集故事板**：不要依赖图像模型生成大量小字。影视分镜信息图、CUT 表或制作设定板这类文字密集故事板，优先先生成 CUT / 场景 / 调度原子图，再按 `script-composed-storyboard.md` 用脚本拼成 4K 故事板母图。

### 不使用故事板时

没有故事板母图时，视频 prompt 必须更详细。先从已确认镜头清单转换，不要直接从自然语言创意跳写；用分镜化结构把模型需要的画面、镜头、声音、人声和文字都写清楚。可能有多个分镜，每个分镜都要独立说明持续时间和音画文字要求。

推荐结构：

```json
{
  "视频内容": {
    "剧情总览": "当前所有镜头的完整剧情梗概，作为全局提示词，帮助视频模型保持人物、情绪、风格和动作一致。",
    "镜头数量": 1,
    "技术规格": {
      "画质修饰": "例如：8k resolution, Unreal Engine 5 render, hyper-realistic, cinematic details.",
      "画面风格": "例如：写实风格、胶片颗粒感、动漫风格，按剧本选择。",
      "参考资产融合": "说明参考图/角色卡/场景卡/概念卡如何影响人物、服装、空间、道具和风格。"
    }
  },
  "shots": [
    {
      "分镜序号": 1,
      "视觉内容": {
        "画面描述": "极度详细的画面描写，包含环境细节、人物站位、动作、微表情，并融合概念卡特征。",
        "画面氛围": "压抑 / 温馨 / 孤寂 / 赛博朋克等心理感受。",

        "运镜方式": "缓慢推镜头 Dolly In、希区柯克变焦 Dolly Zoom、手持跟随 Handheld Tracking 等。",
        "持续时间": 4.0
      },
      "音频轨道": {
        "音乐": "音乐风格、节奏、情绪和进出点；无音乐则写无音乐。",
```

## 字段取舍矩阵

| Scene                       | 必填字段                                       | 条件字段                     | 通常省略               |
| --------------------------- | ---------------------------------------------- | ---------------------------- | ---------------------- |
| Product / Commercial        | 成片目标、主体/产品、卖点动作、镜头设计、限制  | 参考关系、声音设计、动作节奏 | 复杂剧情、角色心理     |
| Short Drama / Narrative     | 成片目标、角色目标、情绪变化、动作节奏、结果   | 对白、参考关系、镜头设计     | 多地点世界观解释       |
| Character Action            | 主体一致性、动作路径、镜头跟随、限制           | 参考视频、声音节拍           | 复杂剧情、过多转场     |
| Atmosphere / Mood Film      | 场景、光线/天气、核心视觉变化、镜头速度        | 声音氛围、参考图             | 多拍分镜、强卖点       |
| Social Short / Viral Hook   | 前 1-2 秒钩子、主体动作、节奏、画幅            | 声音节拍、分镜               | 慢热铺垫、长旁白       |
| Video Edit / Extension      | 保留什么、改变什么、延续到哪里、参考视频角色   | 声音/对白、限制              | 全新场景重写           |
| Reference-Driven Generation | 每个素材编号与用途、主体一致性、输入模式、限制 | 动作节奏、声音设计           | “参考所有素材”笼统表达 |

可以省略的部分：

- 无参考素材时省略“参考关系”。
- 无声视频时省略声音设计或明确“无声”。
- 用户要单一动作时，不强拆复杂三段式。
- 用户要特定场景时，先保留场景边界，不替换成通用广告/电影模板。
- 用户需要控场、参考素材多、限制多时，可用场记板方法组织；用户需要多节拍叙事时，可用分镜方法组织。

## 镜头词库

- 建立镜头：远景、大全景、俯拍、低角度仰拍。
- 角色镜头：中近景、特写、过肩镜头、侧后方跟拍。
- 运动镜头：缓慢推镜、快速拉远、横向平移、环绕、升降、手持轻微晃动。
- 产品镜头：微距特写、棚拍转台、光线扫过、材质细节拉近。
- 情绪镜头：浅景深、焦点转移、慢动作、逆光轮廓、闪回式过渡。

## 声音设计

有声视频先读 `sound-design.md`，再把 audio brief 压进最终 prompt。不要只写“生成声音”，要说明声音承担什么功能：

- 音乐：电子、管弦、钢琴、鼓点、低频脉冲、温暖木吉他。
- 音效：脚步、风声、机械声、水滴、布料摩擦、镜头冲击声。
- 对白：用引号，并标注角色和情绪。
- 字幕/标题文字：对白字幕、旁白字幕、标题文字、品牌字样、UI 文字，或明确不要字幕、标题文字、logo、水印和随机字母。
- 参考音频：明确“参考节奏”“参考音色”“参考情绪”，不要让音频独自作为输入。
- 无声：明确 `generate_audio=false`，并写“不要生成音乐、对白或环境音”。

## 全能参考写法

参考素材要在 prompt 中显式编号。编号按 payload 中同类素材的输入顺序计算：第 1 个 `reference_image` 是 `参考图1` / `图片1`，第 2 个是 `参考图2` / `图片2`；第 1 个 `reference_video` 是 `参考视频1` / `视频1`；第 1 个 `reference_audio` 是 `参考音频1` / `音频1`。这和火山官方示例一致，比笼统写“参考所有素材”稳定。

```text
参考图1用于主角面部和服装，不改变发型和主色。
参考图2用于场景氛围和灯光色彩。
参考视频1用于动作节奏和镜头跟拍方式，不复制具体背景。
参考音频1用于鼓点节奏和紧张情绪，画面动作跟随鼓点加速。
```

## 自检清单

- 是否尊重用户指定场景，而不是强行套公式。
- 5-8 秒视频是否只有一个核心动作或一个清晰转折。
- 主体、镜头、声音是否互相服务。
- 是否写清参考素材用途。
- 是否写清输出规格、声音、说话人声、字幕/标题文字和不需要的输入。
- 是否把风格词堆太多。
- 是否要求模型同时做互斥任务。
- 是否适合当前 API 模式。
