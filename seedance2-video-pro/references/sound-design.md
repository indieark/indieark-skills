# Sound Design Reference

> Load when: 生成任务是有声视频、用户提到音效/音乐/对白/配音/参考音频，或 `--generate-audio true` 将被使用。
> Avoid: 用户明确要无声视频，或只做传统后期混音、配乐剪辑、降噪、音频转码。
> Pairs with: `director.md` 写最终视频 prompt；`api.md` 和 `cli.md` 确认 `generate_audio` / `reference_audio` 边界；`completion.md` 汇报声音设计和验收。

本文件是声音设计的单一运行时入口。不要只写“生成声音”或只打开 `--generate-audio true`；有声视频必须给模型一个可执行的声音方案。

## Capability Boundary

- `generate_audio=true` 只表示请求模型生成有声视频，不等于已经设计好音效。
- `reference_audio` 是节奏、音色、情绪或对白参考，不能单独作为唯一非文本输入，也不能代替画面 prompt。
- 如果用户要求精确配乐、逐帧 Foley、干净可控对白、歌词、口型严格同步或商业级混音，应说明这属于后期音频工作流；本 Skill 只能把声音意图写入视频生成 prompt，必要时建议先生成无声视频再外部配音/混音。
- 如果用户没有明确要声音，默认先判断场景。需要沉浸感、社媒节奏、广告冲击或剧情对白时问 1 个声音问题；低风险草稿可以用简短默认值并说明。
- 字幕、标题文字和画面文字不是 `generate_audio` 的附属项。有声视频可以不要字幕；无声视频也可以有标题文字。生成前必须单独确认文字轨道。

## Audio Brief

有声视频提交前必须形成一个简短 audio brief：

```text
声音策略：
- 模式：有声 / 无声 / 参考音频驱动
- 音乐：<风格、速度、情绪、是否随镜头增强或淡出>
- 环境音：<空间底噪、天气、城市、室内、机械等>
- 动作音效：<与画面动作绑定的 1-3 个关键声音>
- 说话人声：<是否有人声；对白、旁白、口播、喘息/笑声，或明确不要人声>
- 对白/旁白：<角色、台词、情绪；没有则写不要对白或旁白>
- 字幕/标题文字：<是否有画面文字、黑屏字幕、对白字幕、品牌字样；没有则写不要字幕或标题文字>
- 参考音频：<参考音频1用于节奏/音色/情绪/对白，不复制无关内容>
- 禁止：<不要人声/不要夸张音效/不要突兀音乐/不要新增剧情对白>
```

没有某项就删掉，不要为了完整而塞空字段。短视频通常只保留 2-4 条声音指令。

## Route By Scene

| Scene | Useful Sound Design | Avoid |
| ----- | ------------------- | ----- |
| Product / Commercial | 品牌调性音乐、产品触感音、按钮/液体/材质音、短口播 | 复杂剧情对白、抢戏音乐 |
| Short Drama / Narrative | 情绪底乐、关键环境音、1 句以内对白或旁白 | 多角色长对白、歌词化表达 |
| Character Action | 脚步、衣料、武器/道具、呼吸、动作冲击点 | 混乱连续爆音、无节奏噪声 |
| Atmosphere / Mood Film | 风声、雨声、城市远噪、低频氛围、极简音乐 | 强鼓点、解释性旁白 |
| Social Short / Viral Hook | 前 1 秒音效钩子、节拍、转场冲击声 | 慢热铺垫、复杂配器 |
| Reference-Driven | 参考音频节奏/情绪/音色与画面动作绑定 | 只写“参考音频”不说明用途 |

## Prompt Rules

把声音写成可执行的画面同步指令：

- 音乐要有风格和情绪，例如“轻快电子鼓点，前 2 秒弱，产品出现时增强”。
- 环境音要服务空间，例如“雨声和远处城市噪声作为底噪，不盖过动作音效”。
- 动作音效要绑定画面事件，例如“杯子落桌瞬间有清脆玻璃碰撞声”。
- 对白要短，标注角色和情绪；如果不要对白，明确写“不要生成对白/旁白”。
- 参考音频必须编号，例如“参考音频1只用于鼓点节奏和紧张情绪，不复制旋律或人声”。
- 不要在同一个短视频里同时要求复杂音乐、密集 Foley、多人对白和强环境音；优先级冲突会让声音发散。

## Speech And Text Track

<!-- contract: sound-system-manages speech-track subtitle-track title-text-track no-random-text -->

声音系统不只管理音效。生成前必须明确是否包含说话人声和画面文字：

- **说话人声**：对白、旁白、口播、低语、喘息、笑声或明确“不要人声”。有台词时使用 `角色名 (情绪): 台词内容`；无台词时写“不要生成对白/旁白/口播”。
- **字幕**：对白字幕、旁白字幕、黑屏字幕、下三分之一字幕。若没有字幕，明确写“不要字幕”。
- **标题文字 / 画面文字**：品牌字样、章节标题、UI 文字、提示语或片头片尾文字。若不需要，明确写“不要新增标题文字、logo 或水印”。
- **可读性限制**：任何指定文字都必须清晰可读，不乱码，不生成未指定的随机字母、伪字、logo 或水印。
- **故事板例外**：如果故事板母图已经承载字幕、标题或文字说明，`video-prompt.txt` 只需要说明按故事板文字区显示，并禁止新增其他文字。

如果用户说“字母”但上下文像是在说画面文字，优先理解为字幕/标题文字/画面文字，并在高风险生成前确认一次。

## Final Audio/Text Input Lock

<!-- contract: final-audio-text-input-lock no-unneeded-audio-inputs audio-text-independent-tracks -->

生成前必须把声音和文字轨道写成明确选择，不能让模型猜：

- 有声 + 无字幕：写明 `generate_audio=true`，说明音乐/环境音/动作音效，同时写明不要字幕、标题文字、logo、水印或随机字母。
- 有声 + 字幕/标题文字：写明声音方案和指定文字；指定文字必须短、可读，并说明出现位置或时机。
- 无声 + 无字幕：写明 `generate_audio=false`，不要音乐、对白、旁白、环境音、字幕、标题文字、logo、水印或随机字母。
- 无声 + 画面文字：写明 `generate_audio=false`，并单独说明标题文字、字幕卡或 UI 文字。
- 参考音频驱动：只在参考音频有明确用途时提交；prompt 必须写清参考音频编号和用途，不提交无用途音频。

故事板只提供视觉和镜头控制。即使故事板里有声音区或字幕区，`video-prompt.txt` 仍要写清本次最终声音策略和文字轨道；没有字幕或标题文字时也要明确禁止新增。

## Prompt Snippets

产品广告：

```text
声音设计：有声。轻快高级电子节拍作为底乐，产品出现时加入短促的柔和冲击声；按钮按下时有清脆机械点击声。不要对白，不要夸张音效。
```

短剧：

```text
声音设计：低频紧张氛围音乐，室内远处电流声持续；女主低声说“别回头”，语气压抑。不要加入其他角色对白。
```

动作：

```text
声音设计：脚步声跟随奔跑节奏，衣料摩擦和一次金属碰撞作为动作音效；音乐只做低频推进，不要盖过脚步。
```

氛围片：

```text
声音设计：无对白。风声、远处城市噪声和轻微低频氛围铺底，镜头推进时音乐缓慢增强。
```

参考音频：

```text
参考音频1只用于鼓点节奏和紧张情绪，画面动作在重拍处推进；不要复制参考音频中的人声或旋律。
```

无声：

```text
声音设计：无声视频，`generate_audio=false`。不要生成音乐、对白或环境音。
```

## Pre-Submit Check

提交前检查：

- 如果 `generate_audio=true`，prompt 是否包含明确声音设计或参考音频用途。
- 如果用户不要声音，CLI 是否使用 `--generate-audio false`，prompt 是否写明无声。
- 如果有 `reference_audio`，是否同时有图/视频参考，且 prompt 写清 `参考音频1` 的用途。
- 是否已经分别确认声音、说话人声、字幕/标题文字，而不是用一个“有声/无声”字段代替全部。
- 是否避免提交无用途参考音频或无关声音要求。
- 声音设计是否只保留 1 个主策略，避免音乐、对白、环境音和动作音效互相抢焦点。
- 声音是否服务画面节奏，而不是独立讲另一个故事。

## Completion Check

完成回复必须说明：

- 本次是有声还是无声。
- 实际提交的声音设计摘要。
- 是否使用 `reference_audio`，以及它的用途。
- 如果结果音效不理想，下一轮只改 1-3 个声音点，例如音乐风格、动作音效、对白有无、参考音频用途。
