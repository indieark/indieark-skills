# Clapperboard Method

> Load when: 需要单镜头或少量镜头控场，且素材、声音、限制或参数必须清楚。
> Avoid: 用户需要多节拍叙事、短剧转折、广告起承转合；改用 `storyboard.md`。
> Pairs with: `../scenes.md` 的 Atmosphere / Character Action / Video Edit 场景；`../director.md` 的 prompt 字段。

场记板方法把一次生成当作一次短镜头拍摄前的场记板。它适合把镜头目标、素材角色、声音、限制和执行参数压成一张清楚的镜头卡。

## When to Use

Use when:

- 用户指定了具体场景、素材、声音或强限制。
- 需要避免 AI 随意改场景、改主体、改动作。
- 需要把参考图、视频、音频的用途讲清楚。
- 需要一个连续镜头或少量镜头，而不是复杂长剧情。

Avoid when:

- 用户只是在探索创意方向。
- 用户需要多镜头故事板，而不是单镜头执行。
- 用户请求极轻量，只缺一句 prompt。

## Slate Fields

只选必要字段，不要机械全填。

```text
项目/用途：广告、短剧、角色动作、氛围、社媒、编辑/延长。
场景：地点、时间、天气、空间状态。
镜号：单镜头、镜头 A/B，或连续镜头。
主体：角色/产品/物体，以及不能改变的特征。
动作：一个核心动作或变化。
镜头：景别、机位、运镜、速度、焦点。
参考：参考图/视频/音频分别用于什么。
声音：有声/无声、音乐、环境音、音效、对白或参考音频用途。
限制：保持、禁止、不要生成的内容。
参数：画幅、时长、模型、生成有声/无声。
```

## Prompt Pattern

```text
场记板：
用途：...
场景：...
主体：...
动作：...
镜头：...
参考：...
声音：...
限制：...

生成指令：
在上述场记板约束下生成一个...秒...画幅视频。保持...，不要...
```

## Example

User asks:

```text
我要一个非常安静的博物馆夜间巡展镜头，不要广告感，不要夸张运镜，只要月光照着玻璃展柜慢慢移动。
```

Good slate:

```text
用途：氛围片。
场景：夜间博物馆展厅，空间安静，月光穿过高窗照在玻璃展柜上。
主体：玻璃展柜和展品轮廓，不加入人群、字幕、logo。
动作：月光和反射随镜头缓慢移动，展柜边缘出现轻微高光变化。
镜头：单个连续慢速横移镜头，低强度运动，稳定，不做快速推拉或旋转。
声音：无声或极轻环境底噪，按用户偏好确认。
限制：不要广告感，不要夸张运镜，不要把场景改成商业产品展示。
```

## Self Check

- 是否保留了用户指定场景？
- 是否只有一个清晰镜头目标？
- 是否把参考素材用途写清楚？
- 是否没有把场记板当成必须完整输出的表格？
- 是否没有和故事板方法重复堆叠？
