# References

这些文件是 Skill 运行时按需读取的参考，不是仓库维护文档。

加载顺序约定：**入口决策 → 流程与复盘 → 场景与方法 → Provider 边界 → 运行时与产物**。这五组是运行时导航分区；具体前后顺序以 `../SKILL.md` 的「决策管线」为准。遵守渐进披露：先读入口，再按任务链路加载必要文件；不要因为在同一分区就批量加载整组。

## 入口决策（Entry & Routing）

判断是否追问、选哪条任务路线。

| File             | Load When                                                      |
| ---------------- | -------------------------------------------------------------- |
| `interaction.md` | 需要判断是否问询用户、给方向选项、或停止追问直接执行           |
| `guide-mode.md`  | 用户要功能介绍、使用向导、支持模式、准备事项或边界说明         |
| `advisor-mode.md` | 用户要参考包 / 自己去 Sora\|Veo3\|Kling\|Runway 等其他平台跑 / 只要 prompt / 不要生成 |
| `router.md`      | 复杂请求需要判断追问/执行、输入模式、场景、导演方法和 API 操作 |

## 流程与复盘（Workflow & Review）

把自然语言创意推进到剧本、资产、故事板、生成前确认和结果复盘。

| File                    | Load When                                                      |
| ----------------------- | -------------------------------------------------------------- |
| `generation-workflow.md` | 需要阶段提醒、素材清单、标准文本回复、确认门、结构化修改意见或结果复盘 |

## 场景与方法（Scenes & Craft）

把用户意图收束到具体场景 → 选择导演方法 → 渲染成 prompt。

| File                | Load When                                                      |
| ------------------- | -------------------------------------------------------------- |
| `scenes.md`         | 用户指定具体场景、用途或素材组合，需要避免强套通用模板         |
| `methods/README.md` | 需要选择场记板、分镜等导演方法（方法分开存储，仅选一个主结构） |
| `visual-assets.md`  | 复杂视频任务需要规划人物设定图、场景设定图、故事板母图、脚本拼版故事板、版式变体、智能编排、自动拆分或多段连续性；二级索引见 `visual-assets/README.md` |
| `visual-storyboard.md` | 复杂任务需要生成前资产检索、项目资产库、视觉故事板、确认门和生成历史 |
| `sound-design.md`   | 有声/无声、音效、音乐、说话人声、字幕/标题文字、画面文字或参考音频需要形成可执行方案 |
| `director.md`       | 需要写导演 prompt、澄清用户意图、选择镜头/动作/声音            |

火山方舟 API 调用契约 + CLI 命令模板。

| File     | Load When                                                      |
| -------- | -------------------------------------------------------------- |
| `api.md` | 需要修改或确认火山方舟 API 调用、payload、模型、端点、互斥规则 |
| `cli.md` | 需要写 `scripts/seedance2_video.py` 的命令行调用模板           |

## 运行时与产物（Runtime & Artifacts）

完成回复、证据归档和反馈驱动的迭代修正。

| File           | Load When                          |
| -------------- | ---------------------------------- |
| `completion.md` | 生成完成、失败、超时或下载后，需要规范回复用户 |
| `iteration.md` | 用户反馈结果不满意，需要定向修正   |

## 维护约定

- 每次任务只加载真正需要的文件，遵守渐进披露。
- 保持本目录精简。超过 100 行的新增参考应有目录或拆分，并从 `SKILL.md` 与本表同时登记。
- 每个子目录只能有一个「子目录入口」（如 `visual-assets.md` 之于 `visual-assets/`），子目录 README 只承载二级索引，不重复运行时入口的内容。
