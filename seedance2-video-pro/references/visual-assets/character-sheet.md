# Character Sheet Prompt Method

> Load when: 需要生成或修正人物设定图、三视图、多视图角色图，用于后续视频角色一致性。
> Avoid: 用户只需要场景、产品图、故事板母图或已有稳定人物图。
> Pairs with: `setting-director.md` 提供人物设定；`scene-sheet.md` 提供空间；`seedance-handoff.md` 登记和引用。

人物设定图用于锁定角色身份，不追求复杂动作。它应让后续故事板和视频段都能复用同一张角色锚点。

本文件的硬规则：正式角色卡提示词只能使用下面两种之一：`人设三视图提示词` 或 `多视图人设提示词`。第一种 `人设描述提示词` 只作为两种角色卡的文本基础，不单独作为复杂视频项目的最终角色卡；快速草稿可按用户明确要求使用单张半身或全身图。

## Required Inputs

- 角色名和剧情功能。
- 年龄感、身份、气质。
- 身高体型、姿态、比例。
- 脸部 DNA：脸型、眼睛、鼻子、嘴巴、眉毛、肤色、妆容。
- 发型和发色。
- 服装颜色、款式、材质和状态。
- 固定配饰或道具。
- 项目统一风格。

缺少脸、发型、服装或识别点时，先补齐或问用户，再生成。

## Confirm Before Generation

生成或调用外部图像工具前，先给用户确认一份短包：

```text
人物设定图确认：
- 角色：
- 用途：主角外貌锁定 / 配角设定 / 反派设定 / 三视图 master / 多视图 master
- 设定图类型：three-view / multi-view
- 输出尺寸：至少 2K；细节丰富、高一致性或主角 master 优先 4K
- 脸部 DNA：
- 发型：
- 服装和材质：
- 体态比例：
- 固定识别点：
- 一致性要求：
```

用户确认后再生成图片。生成结果不符合脸、发型、服装、体态或识别点时，先修人物设定图，再进入场景或故事板。

## 第一种：人设描述提示词

这是三视图和多视图角色卡后半段要粘贴的完整具体描述，不是复杂项目的最终角色卡图片提示词。它必须写到足够具体，用来保证后续视图互相一致。

```text
人设描述提示词：
角色：<角色名>，<剧情功能/身份/年龄感/气质>。
身高体型：<身高>，<体型>，<肩颈比例>，<四肢比例>，<站姿气质>。
脸部：<脸型>，<眼睛>，<眉毛>，<鼻子>，<嘴巴>，<人脸特写识别点>。
肤色妆容：<肤色>，<妆容>，<疤痕/痣/眼袋/唇色等>。
表情气质：<常态表情>，<气质关键词>。
发型：<长度/颜色/刘海/发髻/发饰/发质>。
穿搭：<上装>，<下装>，<外套/披风/制服>，<鞋靴>。
配饰和道具：<耳饰/项链/手表/玉佩/武器/产品配件等>。
材质颜色：<服装面料>，<金属/皮革/织物/透明材质>，<主色/辅色>。
光影风格：<项目风格 bible>，<中性设定光或指定影视光线>。
一致性要求：五官、身高、体型、发型、服装、配饰、材质、颜色和识别点在所有视图中保持一致。
```

## 第二种：人设三视图提示词

用于普通角色卡和跨镜头稳定。提示词必须以指定句子开头，后接第一种 `人设描述提示词` 的完整具体描述。

```text
16:9 横版构图，角色严格按参考图。左 1/3 高清人脸图，右 2/3 排布正侧背三视图（正面、侧面、背面），纯白背景。比例一致。输出尺寸至少 2K，16:9 不低于 2560x1440；细节丰富或主角 master 优先 4K，推荐 3840x2160。

<粘贴第一种「人设描述提示词」的完整具体描述>

三视图一致性要求：三视图中的身高、体型、五官、发型、穿搭、配饰、材质、颜色、光影完全一致，画面清晰完整。
画面内容：仅展示该角色的中性设定视图，纯净背景，完整身体轮廓，适合后续建模和视频参考。
尺寸要求：普通角色卡最低 2K；需要服饰纹理、道具、脸部识别点或后续大量复用时优先 4K。
```

## 第三种：多视图人设提示词

用于高一致性项目、主角、系列内容或需要后续大量分镜/CUT 复用的最高档角色卡。这里的“多视图”是完整角色设定板：左侧放角色信息和主头像，中间放正面、侧面、背面三个全身视图，右侧放服饰和饰品拆解，底部放发型展示和表情排列。按以下模块完整生成：

```text
多视图人设提示词：
- 画面布局：16:9 横版角色设定板，白色或浅灰干净背景，细灰分隔线和测量参考线，清晰栏目标题。纵向分为上方主体设定区和底部完整展示行；上方主体设定区横向分为左侧信息栏、中央三视图区、右侧拆解栏。整体信息密集、间距清楚，类似专业动画角色设定图、角色 design sheet、turnaround model sheet。
- 输出尺寸：最低 2K，16:9 不低于 2560x1440；多视图 master、主角、系列内容、服饰/饰品拆解复杂或文字标签较多时优先 4K，推荐 3840x2160。正式角色卡使用原生清晰输出图。
- 版面占比：上方主体设定区占画面高度 72%-78%；底部完整展示行占画面高度 22%-28%、画面宽度 100%。上方主体设定区内，左侧角色信息栏占画面宽度 22%-25%，中央三视图区占画面宽度 50%-55%，右侧拆解栏占画面宽度 22%-25%。右侧拆解栏上方 55%-65% 放服饰拆解，下方 35%-45% 放饰品和道具拆解。三视图保持最大视觉主体和清晰完整全身轮廓。
- 左侧角色信息栏：左上放大标题 `<角色名> 角色设定图`，标题区域占左栏高度 10%-12%；下方主头像或半身主视觉占左栏高度 34%-42%，展示常态脸、发型、发饰和上半身穿搭；再下方字段表占左栏高度 30%-36%，列出姓名、代称、发色、发型、眼瞳、脸部特征、唇色和风格关键词；底部角色小传占左栏高度 10%-16%，只写 2-4 行短句。图内文字使用短标签，关键设定写在提示词正文。
- 角色描述：粘贴第一种人设描述提示词，必须包含身高、体型、穿搭、发型、配饰、人脸特写、肤色、妆容、表情和气质。
- 中央主体三视图组：同一人物的正面视图、侧面视图、背面视图，共 3 个全身视图，三个全身人物各占中央三视图区宽度约 30%，人物间留 3%-5% 间隔；站姿中性，手臂自然下垂或轻微外展，脚底对齐；背景有头顶、下巴、肩线、胸线、腰线、臀线、脚底等浅灰水平参考线；三个视图的身高、头身比、肩宽、腰臀比例、腿长和鞋底高度完全一致。每个视图下方用短标签标注正视图、侧视图、后视图。
- 右侧服饰拆解组：位于右侧拆解栏上方 55%-65% 高度，用 4-6 个等宽方形小框拆解上装、下装、外套、鞋袜、关键纹样、包具、武器或手持道具；每个小框显示近距离材质、缝线、图案、扣件、边缘、磨损和颜色层次。复杂服饰优先拆上装领口、袖口、腰部、裙摆或裤脚、鞋底。
- 右侧饰品拆解组：位于右侧拆解栏下方 35%-45% 高度，用 3-5 个方形小框拆解耳饰、发饰、项链、手环、头带、徽章、挂件、手机、包链等固定识别点；饰品和主体三视图中的位置、颜色、大小一致，只展示角色描述里的固定饰品。
- 底部完整展示行：横跨画面宽度 100%，分为左侧发型展示区和右侧表情设定区；两区之间有清晰分隔线，所有缩略图使用统一尺寸方框或细线边框承载，方框之间保留 2%-3% 的可见间隙，整体占满底部展示行。
- 底部发型展示组：固定 5 张完整头部图，每张都包含完整头部轮廓、脸部、颈部上端、头发和发饰；排布正面、45 度侧面、侧面、背面、发型细节五个角度。重点展示刘海、侧发、后发、发饰、发束层次和发色渐变，所有头部比例和发饰位置保持一致。
- 底部表情设定组：固定 6 张完整头部图，每张都包含完整头部轮廓、脸部、颈部上端、发型和发饰；排布常态、威严或冷酷、沉思、温柔、愤怒、果断六个表情。每个表情以眉眼嘴和轻微头部角度变化表达，脸型、瞳色、发型、妆容和年龄感保持一致。
- 文字与标注要求：使用大标题、栏目名和短标签；关键人物设定写在提示词正文；图内标签简短、清晰、服务分区识别。
- 全图要求：同一角色，五官、身高、体型、服装、配饰、光影、材质 100% 完全一致；画面比例 16:9，高精度，线稿干净，色彩准确，适合后续故事板、CUT 图和 Seedance 视频参考。
```

## Character Continuity Tokens

生成视频 prompt 或故事板 prompt 时，重复这些锚点：

```text
角色锚点：<脸型>、<眼睛>、<发型>、<服装主色和材质>、<固定配饰>、<体态比例>。
状态锚点：<雨水/伤痕/道具位置/产品佩戴状态> 从上一镜延续。
```

视频模型需要反复读取可见特征；写 prompt 时重复角色锚点和状态锚点。

## Failure Diagnosis

| Symptom | Fix |
| ------- | --- |
| 换脸 | 增加脸部 DNA，使用三视图或多视图 |
| 换衣 | 固定服装颜色、材质、层次和配饰 |
| 年龄漂移 | 写年龄感、体态比例和妆容 |
| 多图不一致 | 先选一张 master character sheet，再用它派生 |
| 角色抢错主体 | 在 prompt 写“只有该角色为主体”，限制其他人入画 |

## Registration

生成完成后：

```bash
seedance2 asset add <project-id> --type character --name <角色名> --file <image>
```

登记时 `purpose` 写清“主角外貌锁定”“反派设定”“三视图 master”等用途。
