# Setting Director Flow

> Load when: 用户只有一句想法，或复杂任务需要先拆成角色设定、场景设定、风格统一、段落逻辑和视频提示词。
> Avoid: 用户已经提供完整设定且只需要某张图的绘图提示词。
> Pairs with: `character-sheet.md` 生成人物图；`scene-sheet.md` 生成场景图；`multi-segment-continuity.md` 管多段连续性；`storyboard-board.md` 把视频关键信息压缩成故事板母图。

设定导演流是母系统。它不只输出视频 prompt，而是输出一组可登记、可确认、可复用的创作资产。

## When to Use

Use when:

- 用户只给一句短剧、广告或产品创意。
- 任务要求角色一致性、场景一致性或系列内容复用。
- 后续需要生成多张图片资产或多个视频段。
- 用户想法抽象，例如“高级感”“燃”“悬疑”“童话反转”。

Avoid when:

- 用户明确只要快速抽样。
- 用户已经给了完整角色图、场景图和视频 prompt。
- 任务只是查询、下载或改 CLI 参数。

## Output Contract

设定导演流应输出：

1. 需求理解：用户原始想法、用途、平台、画幅、时长、声音取向。
2. 默认补全：缺省时短视频默认 9:16，单段 4-15 秒，1 个主事件。
3. 故事蓝图：一句话故事、核心冲突、情绪曲线、收尾钩子。
4. 视觉圣经：风格、色彩、光线、镜头语言、材质和限制。
5. 人物设定：主角、配角、反派的外貌、服装、道具、动作习惯。
6. 场景设定：主要空间、入口出口、光源方向、关键道具、天气和材质。
7. 段落逻辑：每个 Seedance 生成段的目标、起点、推进、转折、收尾和镜头边界。
8. 连续性台账：多段项目每段的 `start_state`、`end_state`、`carry_over`、`continuity_mode` 和承接素材。
9. 资产清单：需要生成或登记哪些人物图、场景图、故事板母图，以及是否需要九宫格、四栏或制作设定板版式。

## Style Bible

风格规则必须把抽象词翻译成可见元素：

```text
风格：<题材/平台>，<画幅>，<主色调>，<光线方向>，<材质>，
<镜头速度>，<景深/颗粒/质感>，<声音或节奏倾向>。
```

示例：

```text
9:16 竖屏短剧，暗童话悬疑，低饱和红与冷绿，门内暖光和门外冷光对比，
木门粗糙纹理，浅景深，缓慢推近，轻微胶片颗粒，安静到危险逼近。
```

## Character Plan

每个重要角色至少记录：

- 身份：年龄感、职业/关系、剧情功能。
- 身体：身高、体型、姿态。
- 脸部：脸型、眼睛、鼻子、嘴巴、眉毛、肤色、妆容。
- 服装：颜色、材质、层次、破损/雨水/血迹状态。
- 发型：长度、颜色、刘海、发髻、饰品。
- 道具：固定物件、佩戴位置、材质、是否贯穿全片。
- 一致性声明：跨图、跨镜头必须保持哪些要素。

输出后，人物图提示词交给 `character-sheet.md`。

## Scene Plan

每个主要场景至少记录：

- 场景编号和名称。
- 地点、时间、天气、时代感。
- 空间 DNA：入口、出口、门窗、桌面、道路、远中近层次。
- 光线连续性：主光方向、色温、阴影、反射。
- 道具追踪：产品、信物、杯子、门把手、武器等的位置和状态。
- 镜头可行性：角色从哪里进入，镜头如何移动。

输出后，场景图提示词交给 `scene-sheet.md`。

## Segment Plan

Seedance 单段控制在 4-15 秒。复杂剧情要拆段：

```text
SD01：开场钩子，建立主体和危险/卖点。
SD02：冲突推进或功能展示。
SD03：反转、结果或悬念。
```

每段只保留一个核心动作或一个清晰转折。多人、多地点、强对白、复杂打斗要主动压缩或拆段。拆段时优先按镜头边界切开：尽量不要把同一个镜头拆到两次生成中；如果确实是长镜头超过可控时长，再转入 `multi-segment-continuity.md` 判断是否用 `extend` 或尾帧硬衔接。

多段项目还要写可见交接，不要只写剧情摘要：

```text
SD01 end_state: <人物位置 + 道具状态 + 场景光线 + 情绪/动作落点>
SD02 continuity_mode: <cut_continuity / tail_frame_handoff / first_last_bridge / extend>
SD02 start_state: 承接 SD01 end_state；是否必须画面接尾帧由 continuity_mode 决定。
SD02 carry_over: <需要带入 SD03 的服装、道具、伤痕、产品状态或空间变化>
```

如果后一段只是剧情连续但已经切镜头，通常只承接状态、角色卡、场景卡和风格规则，不强制尾帧。只有需要画面硬衔接时，才考虑上一段尾帧作为下一段首帧；同一长镜头继续时优先用 `extend`。

## Self Check

- 是否输出了后续图像生成所需的具体可见信息？
- 是否避免只写“高级感/电影感”等抽象词？
- 是否把角色、场景、风格和段落逻辑分开，方便登记和回溯？
- 是否没有承诺 CLI 会生成图片？
