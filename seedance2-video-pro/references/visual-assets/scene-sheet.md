# Scene Sheet Prompt Method

> Load when: 需要生成或修正场景设定图，用于锁定空间结构、光线方向、道具和视频镜头路径。
> Avoid: 用户只需要人物图、故事板母图或已有稳定场景图。
> Pairs with: `setting-director.md` 提供场景 DNA；`character-sheet.md` 提供人物；`storyboard-board.md` 使用场景图，`grid-storyboard.md` / `four-column-storyboard.md` 可作版式参考。

场景设定图用于让视频模型知道“这个空间怎么拍”。它不是氛围海报，也不是堆满装饰的概念图。

本文件的硬规则：场景卡必须先列主要场景，再逐个主要场景做路由判断，最后生成提示词。不要把多个空间混在一条提示词里；同一场景需要多个角度时，可以拆成多条角度提示词，也可以生成正交四视图或多视角联合图；但空间结构、关键道具、光源方向、天气、材质和色调必须一致。

## Scene Card Routing Contract

<!-- contract: scene-card-routing-required no-default-simple-scene-card scene-asset-density-decision shot-angle-count-gate reuse-strength-gate -->

生成任何场景图提示词前，必须先输出“场景卡路由判断”。不要默认只生成一张简单场景单图；只有在镜头需求和复用强度都支持时，才选择基本场景卡。

```text
场景卡路由判断：
- 场景：<SCxx_场景名>
- 关联镜头：<shot_id 列表>
- 镜头角度数：<单角度 / 多角度：全景、正反打、俯视、道具特写、主观视角等>
- 运镜需求：<静止 / 推进 / 拉远 / 轻微摇移 / 多角度切换 / 复杂调度>
- 复用强度：<一次性 / 本段复用 / 多段复用 / 系列复用 / master 场景>
- 空间风险：<低 / 中 / 高>，说明门窗、入口出口、道具位置、镜头轴线或人物站位是否容易穿帮
- 选择类型：<快速室内外场景 / 基本场景卡 / 单一场景概念图 / 正交四视图 / 多视角联合图>
- 不选其他类型的原因：<为什么不是简单单图 / 为什么不是正交四视图 / 为什么不是多视角联合图>
- 登记计划：<登记为 SCxx master / SCxx angle A / SCxx multi-view board / 裁切为子场景资产>
```

路由规则：

- 单角度、单段、低复用、只有静止/推进/拉远/轻微摇移时，才用基本场景卡或单一场景概念图。
- 有全景定场、核心交互区、正反打、俯视调度、道具特写、主观视角或多个镜头角度时，优先多视角联合图，不要只出单张基本场景图。
- 同一空间会跨多段、系列、多个项目或长期复用时，优先正交四视图作为 master 场景卡；之后可按本次剧情追加多视角联合图。
- 快速室内/室外纯净场景只用于探索或一次性背景；一旦进入故事板或视频生成，必须升级为正式场景卡并补齐路由判断、场景 DNA 和禁止穿帮提醒。
- 如果路由判断选了基本场景卡，但后续镜头拆分增加了新角度或复用需求，必须回到本节重新路由。

## Required Inputs

- 场景编号和名称。
- 空间类型：室内 / 室外 / 半开放空间。
- 地点、时间、天气、时代或品牌调性。
- 空间结构：入口、出口、窗户、桌面、道路、门、走廊。
- 前景、中景、背景层次。
- 主光源方向、色温、阴影和反射。
- 关键道具的位置和状态。
- 角色可能的运动路径或镜头路径。
- 画面限定：无人物、建立镜头、广角或正交视图、无文字水印。

## Confirm Before Generation

生成或调用外部图像工具前，先给用户确认一份短包：

```text
场景设定图确认：
- 场景：
- 路由判断：镜头角度数 / 复用强度 / 空间风险 / 选择类型 / 不选其他类型的原因
- 用途：快速纯净场景 / 基本场景卡 / 单一氛围场景 / 主场景空间锁定 / 产品广告桌面场景 / SC01 正面角度 / 正交四视图 / 多视角联合图
- 空间类型：室内 / 室外 / 半开放空间
- 空间 DNA：
- 入口/出口/门窗：
- 主光方向和色温：
- 关键道具位置和状态：
- 角色或镜头路径：
- 画面限定：无人物 / 建立镜头 / 广角 / 正交 / 无文字水印
- 禁止变化：
```

用户确认后再生成图片。生成结果如果空间结构、光源或道具位置不稳定，先修场景设定图，不要继续做故事板。

## 主要场景列表

复杂项目先列主要场景，只列主要空间，不把角度当成新场景：

```text
主要场景列表：
- SC01_雨夜长街：<一句话说明用途>
- SC02_破庙内殿：<一句话说明用途>
- SC03_屋顶天台：<一句话说明用途>
```

多个主要场景必须一个个生成场景提示词。比如 `SC01_雨夜长街` 和 `SC02_破庙内殿` 是两张场景卡；不要写成“雨夜长街和破庙内殿组合场景”。

## 场景卡类型选择

根据复用强度选择场景图类型，不要把所有场景都做成同一种图：

| Need | Use | Output |
| ---- | --- | ------ |
| 快速抽卡或一次性背景 | 纯净室外/室内场景 | 一张无人物建立镜头 |
| 单个场景里没有切换镜头，只是静止、推进或拉远 | 基本场景卡 / 单一场景概念图 | 一张带时间、氛围、画风和空间 DNA 的场景图 |
| 需要多次复用、建模或严格空间锁定 | 正交四视图 | 2x2 正交图，前/后/左/右或正/反/俯/细节 |
| 某一次剧情场景里会使用多个明确镜头角度 | 多视角联合图 | 2x2 或 3x3 镜头联合图，可裁切为本次剧情的子场景资产 |

这张表不能跳过。每个 `SCxx` 都要先根据镜头角度数、复用强度和空间风险做选择，再写提示词。

室内/室外不是两套互斥方法，只是 `空间类型` 和 `场景 DNA` 的字段。室内重点锁门窗、墙面、家具、灯具和可拍摄路径；室外重点锁地形、道路、建筑关系、天气、天光和远中近层次。

### Basic Scene Card Route

如果一个场景段落没有移动到新空间，也没有正反打、俯视、主观视角、角落特写等镜头切换，只是静止镜头、缓慢推进、缓慢拉远、轻微摇镜或轻微横移，基本场景卡就够了。此时不要默认升级到正交四视图或多视角联合图。

基本场景卡必须写清：

- 主要场景编号和名称。
- 空间类型、地点、时间、氛围和画风。
- 核心道具和摆放位置。
- 主光方向、远中近层次和镜头可移动路径。
- 画面限定：无人物、建立镜头、无文字水印。

### Orthographic Reuse Gate

正交四视图是严格复用型场景卡，只在下列情况使用：

- 同一场景会跨多个视频段、系列内容或多次项目复用。
- 后续故事板、CUT 图或视频段需要反复引用这个空间。
- 需要锁定前后左右关系、门窗位置、家具/道路布局、道具坐标或建模参考。
- 用户明确要求“正交视图 / 四视图 / master scene sheet / 可复用场景卡”。

如果只是一次性镜头推进或拉远，不生成正交四视图。

### Multi-View Shot Gate

多视角联合图服务“某一次具体剧情场景会用到的镜头组合”，不是通用 master 场景卡。只有当本段剧情确实包含多个不同镜头角度时才使用，例如全景定场、核心交互区、补充角落、道具特写、正反打或俯视调度。

多视角联合图必须绑定到具体剧本段落或生成轮次：

- 写清本次剧情段落和镜头列表。
- 每格都是同一场景 DNA 下的一个镜头角度。
- 生成后可裁切为本次剧情的子场景资产。
- 不把多视角联合图当作长期复用 master；长期复用仍走正交四视图。

## Time And Mood Tokens

生成场景概念图时可把中文选择转换为英文风格锚点，帮助外部图像工具稳定画面：

```text
时间光影：
- 白天：daytime, bright sunlight
- 夜晚：nighttime, moonlight, stars
- 黎明：dawn, early morning light
- 黄昏：dusk, golden hour
- 阴天：overcast sky, soft diffused light
- 暴风雨：stormy weather, dark clouds

地点氛围：
- 平静：peaceful, serene
- 紧张：tense, suspenseful
- 浪漫：romantic, intimate
- 神秘：mysterious, foggy
- 欢快：cheerful, vibrant
- 忧郁：melancholic, somber
- 史诗：epic, majestic
- 恐怖：horror, creepy
```

## Quick Indoor / Outdoor Draw

快速场景抽卡用于早期探索或一次性背景。它可以比正式场景卡轻，但仍必须写清地点、物件、时间、氛围、画风和画面限定。

```text
生成纯净场景图片。
空间类型：室外。
地点描述：<平地/山地/城市/村庄/海边/废土/其他具体环境>。
场景物件：<剧情需要的道具；若为空，允许根据地点自行创意，但不要加入人物>。
时间光影：<白天/黄昏/夜晚/黎明/阴天/暴风雨>，<对应英文 time token>。
地点氛围：<温馨/恐怖/孤寂/科幻/其他>，<对应英文 mood token>。
场景画风：<写实/3D 渲染/动漫/卡通/项目风格 bible>。
画面限定：详细背景，环境概念艺术，建立镜头，电影构图，广角镜头，无角色，无无关文字，无水印，无 logo。
```

```text
生成纯净室内场景图片。
空间类型：室内。
室内空间：<日式卧室/赛博朋克书房/废弃工厂/欧式客厅/其他具体空间>。
室内陈设：<剧情需要的家具和道具；若为空，允许根据空间自行创意，但不要加入人物>。
室内布光：<自然窗光/台灯暖光/电视屏幕蓝光/霓虹灯透射/丁达尔效应>。
居住氛围：<生活化/极简/杂乱/温馨/阴森/样板间感>。
场景画风：<写实/3D 渲染/动漫/卡通/项目风格 bible>。
画面限定：室内设计，详细房间，广角镜头，概念艺术，建立镜头，没有角色，无无关文字，无水印，无 logo。
```

快速抽卡成功后，如果这个场景会进入故事板或视频生成，必须把它升级为正式场景卡：补齐场景编号、空间 DNA、角度说明、光线方向、关键道具位置、空间关系和禁止穿帮提醒。

## 场景图提示词格式

每条场景图提示词必须按这个结构写：

```text
<场景编号> + <场景名称>
风格提示词：<项目风格 bible / 影视类型 / 质感>
场景 DNA：<地点、时间、天气、空间结构、建筑材质、道路方向、门窗位置、入口出口、远中近层次、关键道具>
角度说明：<正面视角 / 反向视角 / 俯视视角 / 角色主观视角 / 其他指定视角>
光线方向：<主光方向、色温、阴影、反射；同场景多角度必须一致>
关键道具位置：<剑、信物、酒杯、手机、血书、产品等的位置和状态变化>
空间关系：<人物站位、镜头轴线、入口出口、左右关系；对话和对峙场景要保持方向稳定>
禁止穿帮提醒：不要改变门窗位置、入口出口、光源方向、关键道具位置、天气、材质和主色调；不要把空间画成另一处地点。
```

## Scene DNA Prompt

```text
SC01_雨夜长街
风格提示词：<项目风格 bible>，电影级场景设定图，真实空间透视，适合后续分镜和视频生成。
场景 DNA：地点 <具体地点>，时间 <时间>，天气 <天气>；空间结构 <道路方向/建筑关系/室内外边界>；建筑材质 <材质>；门窗位置 <位置>；入口出口 <位置>；远景 <元素>，中景 <元素>，近景 <元素>；关键道具 <位置和状态>。
角度说明：<正面视角 / 反向视角 / 俯视视角 / 角色主观视角>，镜头轴线 <方向>，人物可从 <位置> 移动到 <位置>。
光线方向：主光从 <方向> 来，<色温>，阴影 <强/弱>，材质反射 <描述>。同场景所有角度保持这个光源方向。
关键道具位置：<剑/信物/酒杯/手机/血书/产品等> 位于 <位置>，状态 <状态>，后续变化 <变化规则>。
空间关系：人物站位 <位置>，入口出口 <左右/前后关系>，对话或对峙时保持 <轴线/左右关系> 稳定。
禁止穿帮提醒：不要改变空间结构、门窗位置、道路方向、光源方向、天气、材质、主色调和关键道具位置；不要让人物抢主体，不要无关文字、水印、logo。
```

## Single Scene Concept Art

单一场景适合一次性使用、氛围优先、或只有静止/推进/拉远镜头的情况。它比快速抽卡更严密，必须带电影美术指导视角：

```text
任务目标：请作为电影美术指导和 Concept Artist，绘制一张场景概念图。

场景基础信息：
- 场景编号和名称：<SCxx_场景名>
- 地点 Location：<具体地点>
- 核心道具 Props：<剧情需要的道具和摆放区域>
- 时间 Time：<中文时间>，<英文 time token>
- 氛围 Mood：<中文氛围>，<英文 mood token>

视觉风格要求：
- 风格流派：<项目风格 bible / 新海诚动画风格 / 8mm 复古胶片 / 写实电影感 / 3D 渲染等>
- 镜头语言：环境空镜 Establishing Shot，电影级构图，广角镜头，空间层次清楚。
- 负面限制：No People，不要出现任何人物；不要模糊噪点，不要水印文字，不要 logo，不要随机文字。
```

单一场景概念图不能替代长期复用场景卡。若同一场景会反复出现在多个段落或项目中，继续生成正交四视图；若只是在本次剧情里需要多个镜头角度，再生成多视角联合图。

## Orthographic Scene Sheet

正交四视图适合长期复用、需要稳定空间结构、后续要多次进入分镜或视频生成的场景。它优先服务“空间一致性”，不追求戏剧光影；一次性场景、单镜头推进或拉远不需要正交四视图。

```text
任务目标：请作为 3D 建模师和电影美术指导，绘制 <场景名称> 的专业正交视图 Orthographic Scene Sheet。

关键要求：
1. 布局：精确 2x2 网格 Four-Panel Grid。
2. 视角：包含同一空间的正面、背面、左侧、右侧四个视角；如场景更需要，也可替换为正面、反向、俯视、关键交互区。
3. 对齐：所有视角比例严格一致，地平线或室内透视高度对齐。
4. 光照：平光照明 Flat Lighting，无强阴影，方便识别空间结构。
5. 限制：No People，不要人物，不要文字、水印、logo。

分格指令：
- 左上格 Front View：<核心空间/物体> 的正面直视镜头；后方墙面或远景包含 <细节>。
- 右上格 Back View：同一空间的背面直视镜头；相反方向墙面或入口包含 <细节>。
- 左下格 Left Side：同一空间的左侧视图；展示 <右墙/道路/侧面结构/道具位置>。
- 右下格 Right Side：同一空间的右侧视图；展示 <左墙/道路/侧面结构/道具位置>。
```

正交四视图中的四格都必须共享同一场景 DNA、道具位置、材质、色调和空间比例。它可以作为 master scene sheet，后续故事板和视频 prompt 从这里继承空间锚点。

## Multi-Angle Rule

同一场景需要多个角度时，分别标明正面视角、反向视角、俯视视角、角色主观视角等；不要把多个空间混成一条提示词。分别生成：

```text
SC01 正面角度：保持同一空间 DNA，镜头朝向木门，主光从画面左后方进入。
SC01 反向角度：仍是同一木门和窗户位置，镜头从门内看向室外，光源方向不变。
SC01 俯视角度：展示门、窗、桌子和角色移动路径，空间比例与前两图一致。
```

所有角度必须共享：

- 门窗位置。
- 主光方向。
- 关键道具位置。
- 材质和主色调。
- 入口出口和左右关系。

对话和对峙场景必须额外锁定镜头轴线、左右关系和人物站位，避免正反打时方向穿帮。

## Multi-View Scene Board

多视角联合图适合某一次剧情场景里会用到多个镜头角度的情况。它不是故事板母图，也不是 CUT 图，更不是长期复用 master 场景卡；它是一个具体剧本段落的镜头视角集合，生成后可裁切成子场景资产，再分别喂给故事板或视频分镜。

```text
角色设定：你是一位世界级电影美术指导。
任务目标：为剧本设计一张场景多视角联合图。
输出要求：2x2 或 3x3 标准网格，包含同一场景的多个视角。无边框，无文字。

场景档案：
- 场景编号和名称：<SCxx_场景名>
- 地点：<具体地点>
- 时间：<中文时间>，<英文 time token>
- 氛围：<中文氛围>，<英文 mood token>
- 风格：<项目风格 bible>
- 场景 DNA：<空间结构、门窗/道路、入口出口、远中近层次、关键道具>

分格指令：
- 第 1 格：全景定场，广角展示整个场景和主要空间关系。
- 第 2 格：核心交互区，展示角色主要活动或产品主要展示的位置，但不画人物。
- 第 3 格：补充视角，展示另一个重要角落、入口、楼梯、门窗或路径。
- 第 4 格：特写细节，展示材质、道具、光线落点或可互动细节。

禁令：所有格子都不要画人 No People；四个格子的光影、天气、材质、主色调和风格必须完全统一；不要文字、水印、logo。
```

多视角联合图如果要裁切成子资产，登记时为每个裁切图保留同一 `scene_id`，并在 `purpose` 里写清 `SC01 全景定场`、`SC01 核心交互区`、`SC01 特写细节` 等用途。

## Prop Tracking

关键道具要在场景卡里记录位置和状态变化，后续故事板和视频 prompt 继承这些锚点：

```text
道具追踪：
- 剑：SC01 起始在男主右手，SD01 结尾落在石阶左侧，SD02 不得回到手中。
- 信物：SC02 放在供桌中央，冷光照亮，不得随机消失。
- 手机/产品：始终位于右前景桌面，屏幕朝向镜头，不能换颜色或型号。
```

## Product / Commercial Scene

产品广告场景还要锁定：

- 产品放置区域。
- 手部进入画面的方向。
- 背景层次不能抢产品。
- 品牌主色和材质光。
- 产品高光镜头可用的桌面或转台。

```text
产品场景设定图：高端办公桌，银白恒温杯位于右前景，电脑和文件虚化在左后景，
暖咖色自然光从左侧窗户进入，桌面干净，预留手从画面左侧拿起杯子的动作路径。
```

## Scene Continuity Tokens

进入故事板和视频 prompt 时复用：

```text
场景锚点：<空间结构>、<门窗/道路/桌面位置>、<主光方向>、<关键道具位置>、<天气/材质/主色调>。
镜头路径：角色从 <入口> 到 <目标点>，镜头 <推近/横移/跟拍>。
```

不要只写“同一场景”。要重复可见空间锚点。

## Failure Diagnosis

| Symptom | Fix |
| ------- | --- |
| 换地点 | 增加空间 DNA 和关键道具位置 |
| 光线跳变 | 写主光方向和色温，跨图重复 |
| 镜头无法拍 | 写入口、出口、运动路径和留白 |
| 道具乱跑 | 写道具位置和状态变化 |
| 场景抢角色/产品 | 降低背景复杂度，明确主体位置 |

```bash
seedance2 asset add <project-id> --type scene --name <场景名> --file <image>
```
