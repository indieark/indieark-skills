# Character Sheet Prompt Method

> Load when: 需要生成或修正人物设定图、三视图、多视图角色图，用于后续视频角色一致性。
> Avoid: 用户只需要场景、产品图、故事板母图或已有稳定人物图。
> Pairs with: `setting-director.md` 提供人物设定；`scene-sheet.md` 提供空间；`seedance-handoff.md` 登记和引用。

人物设定图用于锁定角色身份，不追求复杂动作。它应让后续故事板和视频段都能复用同一张角色锚点。

本文件的硬规则：正式角色卡提示词只能使用下面两种之一：`人设三视图提示词` 或 `多视图人设提示词`。第一种 `人设描述提示词` 只作为两种角色卡的文本基础，不单独作为复杂视频项目的最终角色卡；快速草稿可按用户明确要求使用单张半身或全身图。

## 形态档位（从轻到重）

人物资产是一条从轻到重的三档梯度（概念 SSOT 见 `../../video-gen-pro/references/concepts.md`）：

| 档位 | 形态 | 对应提示词 | 选档判据 |
| ---- | ---- | ---------- | -------- |
| L0 轻 | 单图草稿（非正式） | 单张半身或全身图，可借第一种人设描述作底料 | 龙套或只出现一次的角色；仅探索、一次性、用户明确要求草稿。可用单图，甚至带背景的人图都行，不必纯白底。不作复杂项目最终角色卡 |
| L1 中 | 三视图 | `第二种：人设三视图提示词` | 普通角色卡、跨镜头稳定；正式角色卡的起点。细节较少、无表情变化（左 1/3 只取常态表情）。**无特殊演出需求时已够用** |
| L2 重 | 设定板（多视图）+ 必配面部特写单图 | `第三种：多视图人设提示词` | 主角、系列、高一致性、大量分镜/CUT 复用的 master。最高控制：**必须额外捆绑高清头肩胸/面部特写单图**（做几张典型表情、可跨镜头复用）锚定面部。无此需要时退回 L1 |

正式角色卡从 L1 起，L0 是非正式草稿档；第一种人设描述提示词是 L1/L2 的文本底料，不是独立图档。三档是**按需选档**：龙套用 L0，常规演出用 L1，主角/高一致性/有面部精度要求时才上 L2，且 L2 不可单独交付，必须配特写单图。

这条梯度决定一个角色**做**多重；至于实际**喂**视频模型时挑哪几张（主体角色给 master，背景一闪而过的角色给单图，L2 的捆绑特写按镜头挑对应情绪那一张而非整组全喂），见 `../../video-gen-cinematography/references/input/image-input.md` 的 Input Economy。

## Required Inputs

- 角色名和剧情功能。
- 年龄感、身份、气质。
- 身高体型、姿态、比例。
- 脸部 DNA：脸型、眼睛、鼻子、嘴巴、眉毛、肤色、妆容。
- 发型和发色。
- 服装颜色、款式、材质和状态。
- 固定配饰或道具。
- 项目统一风格。

缺少脸、发型、服装或识别点时，先补齐或问用户，再生成。

## Confirm Before Generation

生成或调用外部图像工具前，先给用户确认一份短包：

```text
人物设定图确认：
- 角色：
- 用途：主角外貌锁定 / 配角设定 / 反派设定 / 三视图 master / 多视图 master
- 设定图类型：three-view / multi-view
- 输出尺寸：至少 2K；细节丰富、高一致性或主角 master 优先 4K
- 脸部 DNA：
- 发型：
- 服装和材质：
- 体态比例：
- 固定识别点：
- 一致性要求：
```

用户确认后再生成图片。生成结果不符合脸、发型、服装、体态或识别点时，先修人物设定图，再进入场景或故事板。

## 第一种：人设描述提示词

这是三视图和多视图角色卡后半段要粘贴的完整具体描述，不是复杂项目的最终角色卡图片提示词。它必须写到足够具体，用来保证后续视图互相一致。

```text
人设描述提示词：
角色：<角色名>，<剧情功能/身份/年龄感/气质>。
身高体型：<身高>，<体型>，<肩颈比例>，<四肢比例>，<站姿气质>。
脸部：<脸型>，<眼睛>，<眉毛>，<鼻子>，<嘴巴>，<人脸特写识别点>。
肤色妆容：<肤色>，<妆容>，<疤痕/痣/眼袋/唇色等>。
表情气质：<常态表情>，<气质关键词>。
发型：<长度/颜色/刘海/发髻/发饰/发质>。
穿搭：<上装>，<下装>，<外套/披风/制服>，<鞋靴>。
配饰和道具：<耳饰/项链/手表/玉佩/武器/产品配件等>。
材质颜色：<服装面料>，<金属/皮革/织物/透明材质>，<主色/辅色>。
光影风格：<项目风格 bible>，<中性设定光或指定影视光线>。
一致性要求：五官、身高、体型、发型、服装、配饰、材质、颜色和识别点在所有视图中保持一致。
```

## 第二种：人设三视图提示词

用于普通角色卡和跨镜头稳定。版式为左 1/3 头肩特写 + 右 2/3 三视图；左侧特写画框自头顶至上胸，**下缘止于锁骨以下、胸线以上**（不是半身、不是全身），只取**常态表情**，不做表情变化（表情变化是 L2 的事）。无特殊演出需求时这一档已够用。提示词必须以指定句子开头，后接第一种 `人设描述提示词` 的完整具体描述。

```text
16:9 横版构图，角色严格按参考图。左 1/3 高清头肩特写，画框自头顶至上胸、下缘止于锁骨以下、胸线以上（常态表情，突出脸部识别点、妆容、发型，以及该角色实际拥有的头饰/耳饰/项链等上半身配饰，无相应配饰则只表现脸部与发型），右 2/3 排布正侧背三视图（正面、侧面、背面），纯白背景。比例一致。输出尺寸至少 2K，16:9 不低于 2560x1440；细节丰富或主角 master 优先 4K，推荐 3840x2160。

<粘贴第一种「人设描述提示词」的完整具体描述>

三视图一致性要求：三视图中的身高、体型、五官、发型、穿搭、配饰、材质、颜色、光影完全一致，画面清晰完整。
画面内容：仅展示该角色的中性设定视图，纯净背景，完整身体轮廓，纯图像、无文字标签，适合后续建模和视频参考。这张图作为视频模型参考，不在图内写任何文字。
尺寸要求：普通角色卡最低 2K；需要服饰纹理、道具、脸部识别点或后续大量复用时优先 4K。
```

## 第三种：多视图人设提示词

用于高一致性项目、主角、系列内容或需要后续大量分镜/CUT 复用的最高档角色卡。这里的“多视图”是一张纯图像角色设定板：左 2/3 放正面、侧面、背面三视图，右 1/3 上半放表情头像、下半放服饰和饰品的细节特写。它直接作为视频模型的视觉参考输入，所以全图纯图像、零文字。不再设置左侧文字信息栏，也不再单独排发型展示行。按以下模块完整生成：

```text
多视图人设提示词：
- 画面布局：16:9 横版角色设定板，白色或浅灰干净背景。纯图像，不写任何标题、字段表、栏目名或文字标签。横向分为左 2/3 三视图区和右 1/3 拆解区；右 1/3 上半为表情区，下半为细节特写区。
- 纯图像理由：这张图会作为视频模型的参考图输入，图内文字会被视频模型当成要还原的画面元素，加重心智负担并污染成片。角色名、脸部 DNA、设定字段一律写进提示词正文和资产登记，不画进图里。
- 输出尺寸：最低 2K，16:9 不低于 2560x1440；多视图 master、主角、系列内容或服饰/饰品拆解复杂时优先 4K，推荐 3840x2160。正式角色卡使用原生清晰输出图。
- 版面占比：左侧三视图区占画面宽度约 2/3，右侧拆解区占约 1/3。右侧拆解区上下二等分：上半为表情区，下半为细节特写区。三视图保持最大视觉主体和清晰完整全身轮廓。
- 角色描述：粘贴第一种人设描述提示词，必须包含身高、体型、穿搭、发型、配饰、人脸特写、肤色、妆容、表情和气质。这些只进提示词正文，不写到图里。
- 左侧三视图组：同一人物的正面、侧面、背面三个全身视图，共 3 个，各占左侧区宽度约三分之一，人物间留 3%-5% 间隔；站姿中性，手臂自然下垂或轻微外展，脚底对齐；背景保留头顶、下巴、肩线、胸线、腰线、臀线、脚底等浅灰水平测量参考线用于对齐，这是几何辅助线不是文字；三个视图的身高、头身比、肩宽、腰臀比例、腿长和鞋底高度完全一致。不写“正视图/侧视图/后视图”等文字标签。
- 右上表情组：4 张或 6 张完整头部图，主角或表情戏重的角色用 6 张，配角或造型简洁用 4 张。表情用固定的基本情绪集，保证跨角色结构一致：6 张为快乐、伤心、恐惧、愤怒、惊讶、厌恶（Ekman 六大基本情绪），4 张取其前四（快乐、伤心、恐惧、愤怒），4 张是 6 张的子集。常态不进表情区，由左侧三视图的中性常态承担。每张都包含完整头部轮廓、脸部、颈部上端、发型和发饰；每个表情以眉眼嘴和轻微头部角度变化表达，脸型、瞳色、发型、妆容和年龄感保持一致。
- 右下细节特写组：4 格或 6 格，服饰配饰复杂用 6 格，简洁用 4 格。近距离展示服装材质、缝线、图案、扣件、边缘、磨损、颜色层次，以及耳饰、发饰、项链、手环、头带、徽章、挂件、武器或手持道具等固定识别点；细节的位置、颜色和大小与左侧三视图完全一致，只展示角色描述里的固定服饰和配饰。
- 全图要求：同一角色，五官、身高、体型、服装、配饰、光影、材质 100% 完全一致；画面比例 16:9，纯图像、无文字，高精度，线稿干净，色彩准确，适合后续故事板、CUT 图和 Seedance 视频参考。
```

### L2 必配：高清面部 / 头肩胸特写单图

多视图设定板本身没有特别大范围的面部写真，脸最容易走形。因此 L2 master **必须额外捆绑至少一张高清头肩胸特写或面部特写单图**：它放大突出脸部识别点、妆容、发型，弥补板内表情区头像偏小的不足，让面部识别度可控。这组特写做**几张典型的**即可：常态一张，再按角色加几个典型情绪（如威严、温柔、果断）；它们跟 multi-view master 一起作为该角色的**可复用特写资产**，后续分镜按需调用对应那张，**不必每个分镜/CUT 额外生成**。板内右上表情区保留作情绪范围参考，与这组独立特写单图**共存、各司其职**——表情区给情绪广度，特写单图给面部精度。无此面部精度需求时只用 L1 三视图即可，不必上 L2。

特写单图复用第二种三视图左 1/3 的特写写法（脸部识别点、妆容、发型、实际配饰）和同一裁切线（画框自头顶至上胸，下缘止于锁骨以下、胸线以上），但单独成图、放到最大；画幅按面部/头肩构图取正方形或长方形均可，不强制 16:9，保证高清即可；每张写明对应情绪。这几张典型特写一次做好、一次登记，后续跨镜头、跨段复用，登记见下方 Registration。

生成视频 prompt 或故事板 prompt 时，重复这些锚点：

```text
角色锚点：<脸型>、<眼睛>、<发型>、<服装主色和材质>、<固定配饰>、<体态比例>。
状态锚点：<雨水/伤痕/道具位置/产品佩戴状态> 从上一镜延续。
```

视频模型需要反复读取可见特征；写 prompt 时重复角色锚点和状态锚点。

## Failure Diagnosis

| Symptom | Fix |
| ------- | --- |
| 换脸 | 增加脸部 DNA，使用三视图或多视图 |
| 换衣 | 固定服装颜色、材质、层次和配饰 |
| 年龄漂移 | 写年龄感、体态比例和妆容 |
| 多图不一致 | 先选一张 master character sheet，再用它派生 |
| 角色抢错主体 | 在 prompt 写“只有该角色为主体”，限制其他人入画 |

## Registration

生成完成后：

```bash
videogen asset add <project-id> --type character --name <角色名> --file <image>
```

登记时 `purpose` 写清“主角外貌锁定”“反派设定”“三视图 master”等用途。

L2 master 的捆绑特写单图同样各自用 `--type character` 单独登记，与多视图 master 同属一个角色，`purpose` 标注“面部锚定特写”或“表情特写：<情绪>”，便于提交视频生成时按面部精度或具体情绪检索对应特写。这几张典型特写一次登记后即可跨镜头、跨段复用，不必每个分镜重做。

## 声线（人物附属音频资产）

声线是人物的**附属音频子资产**，跟人物走，不进图像维度表（概念 SSOT 见 `../../video-gen-pro/references/concepts.md`）。它是单一形态——一段代表角色音色的音频样本，没有 L0→L2 梯度。声线用于在 omni 模式下作为 `reference_audio` 锁定角色音色，使同一角色在多镜头、多段视频里说话音色一致。

声线复用人物之外的同一套资产机制，是独立资产类型 `voice`，通过 `--role` 绑定到角色名：

```bash
videogen asset add <project-id> --type voice --name <声线名> --role <角色名> --file <audio>
```

要点：

- `--role` 填**绑定的角色名**，与该角色的 character 资产对应，便于后续按角色检索声线。
- 音频文件复制到资产目录的 `audio/` 子目录（图像类资产进 `images/`，声线进 `audio/`）。
- 外部已有音频用 `--source external` 登记同一来源，不复制文件。
- 提交视频生成时，把声线作为 omni `reference_audio` 输入；音色锚点写进 prompt，说明该角色用哪条声线。
- 声音设计、说话人声、字幕/标题文字和无声策略的完整规则见 `../../video-gen-pro/references/sound-design.md`，本节只负责声线资产的登记与绑定。
