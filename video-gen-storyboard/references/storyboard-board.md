# Storyboard Board Method

> Load when: 需要把角色、场景、镜头、调度、灯光、情绪、声音或色彩压缩成一张视频主参考图，让用户确认并交给 Seedance 运动化。
> Avoid: 用户只要单独的人物设定图或场景设定图，且暂不进入视频故事板。
> Pairs with: `../../video-gen-assets/references/character-sheet.md` / `../../video-gen-assets/references/scene-sheet.md` 提供锚点；`grid-storyboard.md` 和 `four-column-storyboard.md` 只是可选版式；`../../video-gen-assets/references/seedance-handoff.md` 负责登记和视频提示词。

故事板的核心不是“四栏”“九宫格”或“8 镜头”，而是一张按需组织的信息母图。它把视频生成最需要稳定的叙事、镜头和调度信息放进同一张图里，成为 Seedance 的主要运动和镜头参考。版式、栏目和镜头数量服务项目复杂度，不反过来限制创作。

## Core Idea

故事板母图要回答：

- 主体是谁：角色、产品、服装、道具、识别点。
- 在哪里：空间结构、光线方向、关键道具、环境氛围。
- 怎么动：镜头顺序、人物移动、镜头运动、动作接点。
- 怎么拍：景别、焦段感、构图、机位、调度。
- 什么感觉：色彩、灯光、情绪、声音和节奏。

信息简单时，故事板可以只是 4-6 个连续关键画面。信息复杂时，可以成为电影前期制作设定板，包含角色设计区、场景设计区、俯视调度图、分镜故事区、灯光、声音、色彩和摄影说明。故事板母图可以承载详细文本、表格和说明，只要文字清晰、图文一致，并且真正服务视频生成。

先判断故事板服务的是单个生成轮次，还是整个项目的段落规划。15 秒是单次生成上限，不是故事板默认目标。内容不复杂、总时长落在 4-15 秒内、一张图能表达清楚时，可以一张故事板直接服务一次生成。同一角色连续动作但身份一致性重要时，用简稿故事板 + 角色卡即可。多角色、多场景、宣传片、战斗变化或高信息负载时，即使总时长只有 15 秒，也可以拆成多个 4-6 秒故事板和视频段；每段各自有故事板和配套卡片。

故事板母图有两种主要生产方式：

- 单图生成：把版式、画面和必要文字一次性交给外部图像工具生成，适合轻量或中等信息量。
- 脚本拼版：先生成多张 CUT、角色、场景、俯视调度和细节原子图，再用确定性脚本把真实文字、时间码、对白、声音和镜头说明排成 4K 故事板海报。信息密集、中文说明多或用户需要影视制作板确认时，优先读 `script-composed-storyboard.md`。

## Character And Scene Card Anchoring

故事板、分镜图和每个 CUT 关键画面都不能凭空生成。生成故事板前必须先明确本项目要使用的角色卡和场景/背景卡；如果缺卡，先回到 `../../video-gen-assets/references/character-sheet.md` / `../../video-gen-assets/references/scene-sheet.md` 生成或登记，再继续故事板。

故事板也不能被当成单独足够的细节输入。故事板最擅长表达镜头顺序、动作节奏、调度和整体关系，但不一定能把人物五官、服装材质、空间结构、背景道具和关键道具细节都稳定画清。复杂任务提交给 Seedance 时，应把故事板母图、相关角色卡、场景/背景卡和必要道具/产品卡一起作为参考输入，并在 `video-prompt.txt` 中明确每张参考图的用途和主次。

硬规则：

- 每张故事板母图必须声明使用了哪些角色卡、场景/背景卡、道具卡或产品卡。
- 每个分镜画面或 CUT frame 都必须绑定对应的 `character_refs` 和 `scene_ref`；同一段里换场景时，逐 CUT 标清场景卡。
- 生成 CUT 图、九宫格、四栏故事板或脚本拼版原子图时，提示词要显式写“参考角色卡 <id/name>”“参考场景卡 <id/name>”，并继承角色脸部、服装、道具、空间结构、光线方向和关键道具。
- 故事板里的角色设计区和场景设计区可以展示卡片缩略图或摘要，但不能替代原始角色卡/场景卡；原始卡仍是身份和空间的一致性锚点。
- 如果某个 CUT 需要新角色、新服装、新场景或重大光线变化，先更新或新增对应卡，再生成该 CUT；不要让故事板图自行发明。
- 视频生成提交阶段必须同时带上故事板图和本段会出现的角色卡、场景/背景卡；除非用户明确走快速抽样，否则不要只把故事板一张图传给 Seedance。

这条规则同样适用于“先生成多个分镜再拼成一张故事板”的路线。拼版只是排版方式变化，每个分镜图仍必须从对应角色卡和场景卡生成。

## Density Levels

| Level | Use When | Recommended Content |
| ----- | -------- | ------------------- |
| Light | 单段动作、一个卖点、快速确认 | 4-6 个关键画面，角色/场景锚点，读图顺序 |
| Standard | 单段 4-15 秒，需要完整节奏 | 3x3 或自定义镜头网格，景别变化，动作链，结束状态 |
| Project | 短剧、广告、系列内容 | 主视觉、角色/产品细节、场景/情绪、可执行分镜 |
| Production | 信息量高、需要专业前期确认 | 角色设计、场景设计、俯视调度、镜头表、灯光、声音、色彩、摄影说明；文字多时用脚本拼版 |

不要默认追求最复杂。复杂度越高，越要让故事板母图自己承载细节，`video-prompt.txt` 只说明读图规则、运动区域和禁止项，不再完整复述每个格子里的全部文字。

如果复杂度来自多个无关场景或不同角色组，不要单纯升级到更密的 Production 图；优先拆成多个生成轮次。每张故事板只服务一个段落或一个清楚的镜头目标，其他段落的角色、场景和道具不要作为本段视频输入。拆分判断以 `../../video-gen-script/references/multi-segment-continuity.md` 的 `Intelligent Orchestration Gate` 为准。

## Optional Module Library

按任务复杂度选择模块，不要求全部出现：

- 项目标题区：项目名、类型、时长、画幅、限制条件。
- 角色设计区：正面、侧面、背面、表情特写、服装、道具、动作姿态。
- 场景设计区：空间结构、主光方向、关键道具、环境氛围、可拍摄路径。
- 俯视调度区：人物站位、移动箭头、镜头运动轨迹、镜头编号。
- 分镜故事区：连续关键画面、镜头顺序、动作接点、结束状态。
- 灯光与风格区：冷暖、对比、低照度、柔光、材质和整体影调。
- 情绪关键词区：紧张、轻松、浪漫、神秘、冲突、幽默等，只保留短词。
- 声音设计区：环境音、音乐风格、节奏提示；详细声音说明留在文本 prompt。
- 摄影说明区：景别、焦段感、稳定推进、手持、推拉摇移、特写强化。
- 色彩方案区：主色、辅色、点缀色和禁用色。

Production 密度可以接近电影前期制作信息图；Light 或 Standard 密度应删掉不服务视频生成的模块，避免信息噪声。

定每格的镜头语言（摄影说明区的景别/焦段/构图/运镜）和整片风格（灯光与风格区、色彩方案区）时，填法知识与导演法共用——景别（含焦点）见输入指南 `../../video-gen-cinematography/references/input/shot-scale.md`、构图（含机位角度）见 `../../video-gen-cinematography/references/input/composition.md`、运镜见 `../../video-gen-cinematography/references/input/camera-move.md`，风格核心/画风的可挑选模板见 `../../video-gen-cinematography/references/input/style-core.md`、视觉基调见 `../../video-gen-cinematography/references/input/visual-tone.md`，随段落微调的色彩与影调见 `../../video-gen-cinematography/references/input/lighting-tone.md`。故事板法只是把这些填进母图的格子里，填法本身横切两法。

## Resolution And Text Rules

<!-- contract: no-low-res-storyboard-preview first-pass-4k-storyboard-required reconfirm-storyboard-before-handoff -->

- 故事板母图从第一版正式生成开始就必须按 4K 或更高规格生成；默认要求长边不少于 3840 px，或明确写“4K ultra high resolution storyboard board”。如果外部图像工具只能选倍率，选择等效 4K 档。
- 当前工作流不设置低清预览确认门。不要先生成低分辨率故事板给用户确认版式，也不要把低清图登记为故事板资产或提交给 Seedance。
- 如果外部工具或人工草稿已经产生低清图，只能作为未登记的内部参考；必须重新生成或脚本拼版一张 4K / 等效 4K 故事板母图，再进入用户确认、`storyboard add`、`storyboard approve` 或 Seedance handoff。
- 不要把低清图简单放大后当成合格故事板母图。纯 upscale 不能补回缺失语义、乱码小字或错误构图；只有重新生成或重新拼版后的 4K 图本身清晰可读，并通过确认，才可登记为故事板母图。
- 允许在故事板里放详细文本、表格、镜头说明、对白、声音、灯光和情绪节奏。前提是文字可读、字段有用、图文对应，不出现乱码、伪字、废片式无意义文字。
- 详细文本优先放在故事板母图和 `storyboard-prompt.txt` 中；`video-prompt.txt` 只保留区域主次、运动目标、实际读图顺序、声音 brief 和禁止项。
- 文本和画面必须统一：如果表格写“镜头 3 特写”，画面第 3 区也要是特写；如果文字和画面冲突，视频模型会更容易混乱。
- Storyboard 不是越满越好。只有当文本能减少视频 prompt 负担、提升参考图可读性时才放入图中。

## Applicability Route

关键不在工具本身，而在对的场景选择对的方法。故事板不是万能工具，按场景选择：

- 更适合故事板：画面干净、调度明确、镜头顺序清楚、角色/场景一致性重要、用户需要先确认整体执行方案，例如人物进门、产品展示、舞台演出、短剧片段、广告分镜。
- 更适合传统文字提示词或 prompt-heavy 路径：画面极度模糊、抽象、粒子漫天、爆炸烟尘、魔法能量、混乱战场、纯氛围随机效果，且分镜很难稳定画出来时。此时可以只用设定图/首帧/参考视频，再用 `../../video-gen-director/references/director-method.md` 写强提示词。
- 混合路径：角色、场景和镜头节奏用故事板锁住；复杂粒子、爆炸、烟雾、群体运动和音乐节奏用简短视频 prompt 补充。

| Route | Best For | Main Risk | Prompt Strategy |
| ----- | -------- | --------- | --------------- |
| Storyboard-first | 干净、清楚、可规划、需要一致性的动作和镜头 | 图内顺序不清会导致乱读 | 把大量控制信息前置进 4K 故事板；视频 prompt 只做读图指令 |
| Prompt-heavy | 混沌、粒子、爆炸、烟尘、抽象动态或无法画清的随机效果 | 长 prompt 可能和参考图争夺注意力 | 少用故事板，只用设定图/首帧/参考视频加关键动态描述 |
| Hybrid | 主体和镜头可规划，但有局部复杂特效 | 信息过载 | 故事板锁角色/场景/顺序；视频 prompt 只补无法画清的动态 |

## Adaptive Layout Rules

- 栏目按需要定，不固定四栏。
- 镜头数按视频段落定，不固定 8 镜头或 9 格。
- 信息区可以合并或删除；例如没有对白就不要加对白区，没有复杂走位就不要加俯视调度图。
- 可以使用详细文字、表格和栏目标签，但必须 4K 清晰可读，并且字段要和对应画面一致。不要为了装满信息而加入无关说明。
- 对视频最关键的镜头顺序必须视觉上清楚：用网格、编号区位、箭头路径或明确读图顺序表达。
- 高能力图像模型可以直接生成包含中文文字、CUT 表、时间码、对白和复杂版式的完整 4K 故事板母图；提示词要写清文字内容、栏目结构、字号层级、读图顺序、留白和画面对应关系。
- 需要确定性可编辑文字、指定字体/品牌版式、超密集 CUT 表或程序化复盘时，进入 `script-composed-storyboard.md`，让脚本负责最终文字和版式。

## Production Board Prompt Pattern

```text
生成一张 4K 高度精细的电影前期故事板母图，用作后续视频生成的主运动和镜头参考图。
版式根据项目复杂度自适应，不强制四栏或九宫格；信息密集但排版清晰，电影级质感，专业前期制作设定板。
图内中文文字、CUT 编号、时间码、对白和栏目标题清晰可读；文字内容只使用下方字段，并与对应画面区域一致。

项目信息：
项目名称：<title>
类型：<genre>
时长：<duration>
画幅：<ratio>
限制条件：<镜头数量 / 角色数量 / 场景数量 / 其他限制>

参考资产：
角色卡：<character_card_id/name/path>，用于锁定 <角色名> 的脸部、服装、体态、配饰和道具。
场景/背景卡：<scene_card_id/name/path>，用于锁定 <场景名> 的空间结构、背景层次、光线方向、关键道具和环境氛围。
道具/产品卡（按需）：<prop_or_product_card_id/name/path>，用于锁定 <关键道具/产品> 的形状、材质、颜色和状态。
每个分镜画面都必须参考对应角色卡和场景卡，不允许自行改变角色身份、服装、空间或光线逻辑。

角色信息区：
展示 <角色/产品> 的稳定外观、正面/侧面/背面或关键视角、面部/材质特写、服装和道具。
必须保持角色/产品在所有区域一致。

场景信息区：
展示 <场景名> 的空间结构、关键道具、真实光影、环境氛围和可拍摄路径。

调度信息区：
如需要，加入俯视平面调度图，显示人物移动、镜头运动轨迹、关键站位和空间关系。

分镜故事区：
用 <镜头数量> 个连续关键画面表达视频顺序。
每个画面对应：<景别/动作/情绪/转折/结束状态>。

风格信息区：
灯光：<冷暖/对比/主光方向/低照度/柔光等>。
色彩：<主色板>。
摄影：<稳定推进/手持/远景建立/特写强化等>。
声音氛围：<环境音/音乐风格/节奏>。
文字表格：允许加入镜头表、对白、声音、灯光、情绪节奏等详细文本，但必须清晰可读、图文一致。

限制：
不要乱码文字，不要水印，不要杂乱拼贴，不要脱离角色卡或场景卡，不要让角色/产品/场景在不同区域变化。
分镜故事区必须能让视频模型读出明确顺序；其他区域用于锁定人物、场景、风格和调度。
```

## Video Prompt Rule

<!-- contract: storyboard-concise-prompt no-storyboard-detailed-per-shot-prompt speech-track text-track -->

故事板图越完整，`video-prompt.txt` 越要短。不要把故事板里的详细表格逐字复述给视频模型；只告诉模型怎么读图、哪个区域决定运动、哪些区域只做约束：

短不等于省略最终输入控制。故事板只控制镜头内容、动作节奏和调度；`video-prompt.txt` 仍要明确输出规格、参考图主次、声音、说话人声和字幕/标题文字策略。

如果生成有声，`video-prompt.txt` 仍要保留声音 brief：音乐、环境音、动作音效、说话人声/无说话人声、字幕/标题文字策略或参考音频用途；没有声音方案时用 `--generate-audio false`，并写明无音乐、对白、旁白、环境音、字幕、标题文字、logo、水印或随机字母。

这条短提示词规则只适用于故事板路线。没有故事板母图时，不能偷懒写成“按描述生成”；必须回到 `../../video-gen-director/references/director-method.md` 的详细视频生成提示词结构，按分镜写清视觉内容、镜头参数、音频轨道、说话人声、字幕/标题文字和转场。

```text
参考图1是 @故事板母图，用于决定 <时长> 秒 <画幅> 视频的镜头顺序、动作节奏和调度。
参考图2..N 是本段对应角色卡、场景/背景卡和必要道具/产品卡，用于锁定人物、空间、背景层次和关键道具细节。
输出规格：<画幅>，<清晰度/分辨率>，<时长> 秒，<有声/无声/参考音频驱动>。

读图规则：
- 分镜故事区决定视频镜头顺序和动作连续性。
- 角色卡参考图优先用于锁定人物五官、体型、服装、配饰和随身道具，不让故事板里的简化人物覆盖角色卡。
- 场景/背景卡参考图优先用于锁定空间结构、背景层次、光线方向、材质和关键道具，不让故事板里的简化背景覆盖场景卡。
- 调度信息区用于理解人物移动和镜头运动。
- 灯光、色彩、摄影和声音区用于统一风格，不新增剧情。
- 声音与文字：<无声 / 音乐风格 / 环境音 / 动作音效 / 说话人声或无说话人声 / 字幕或标题文字策略 / 不要 logo、水印或随机字母>。

运动目标：
按故事板母图中已写明的镜头顺序，把关键画面连接成连续动作和连续情绪。

保持角色卡中的人物、服装和配饰一致；保持场景/背景卡中的空间、光线和道具一致；按故事板母图执行镜头和运动。
不要把说明区当成新剧情，不要新增故事板之外的角色、场景或结果。
```

## Confirm Before Generation

生成故事板母图前，先确认：

```text
故事板母图确认：
- 故事板用途：主运动参考图 / 项目确认板 / 单段执行板
- 信息密度：Light / Standard / Project / Production
- 必须放进图里的信息区：
- 可以留在文本里的信息：
- 实际镜头数量：
- 已绑定角色卡/场景卡：
- 每个分镜或 CUT 对应的角色卡/场景卡：
- Seedance 提交时会同时输入的角色卡、场景/背景卡、道具/产品卡：
- 读图顺序：
- 哪个区域决定视频运动：
- 哪些区域只用于角色/场景/风格约束：
- 图片规格：首版正式故事板即 4K 或更高
- 图内文字：无 / 简短标签 / 详细表格与说明
- 低清预览：不使用，不登记，不作为确认门
- 4K 复核：镜头顺序、文字可读性、角色/场景卡引用和参考图用途是否一致
- 视频提示词策略：简短读图规则 / 需要补充复杂特效文字
```

用户确认故事板方案后，第一张进入确认和登记流程的故事板图就按 4K 或等效 4K 生成。生成后必须再次让用户确认：这张图是否已经包含本次视频需要的全部关键视觉信息，图内文字是否清晰可读，分镜故事区是否足够指导 Seedance。

## Relationship to Other Methods

- `grid-storyboard.md`：标准单段故事板版式之一，适合镜头顺序优先。
- `four-column-storyboard.md`：项目级故事板版式之一，适合主视觉和分镜同时确认。
- 电影制作设定板：Production 密度的故事板母图，适合复杂短片或系列项目。
- 这些都是故事板，不是不同系统；区别只在信息密度和版式。
