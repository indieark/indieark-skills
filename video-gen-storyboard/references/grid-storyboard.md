# Grid Storyboard Layout

> Load when: 已理解 `storyboard-board.md` 的故事板母图理念，并且单段 4-15 秒视频需要用镜头网格锁定角色、场景、镜头顺序和情绪节奏。
> Avoid: 还没有决定故事板母图的信息密度和区域主次；先读 `storyboard-board.md`。
> Pairs with: `storyboard-board.md` 提供上层概念；`../../video-gen-assets/references/setting-director.md` 提供段落逻辑；`../../video-gen-assets/references/character-sheet.md` / `../../video-gen-assets/references/scene-sheet.md` 提供锚点；`../../video-gen-assets/references/seedance-handoff.md` 进入 Seedance。

九宫格是故事板母图的一种镜头网格版式，不是故事板本身的定义。它适合把一个 Seedance 生成段压缩成连续关键帧，让视频模型主要负责运动化。九格是默认参考结构，不是硬性格数；真实任务可以按场景改成 4-6 个关键瞬间、8 个节拍或更细分的镜头，但必须保持同一段视频的时间顺序。

## When to Use

Use when:

- 一条视频只讲一个动作、一个转折或一个卖点。
- 用户需要控制镜头顺序、景别变化、动作接点和收尾钩子。
- 已有或可以先生成角色设定图、场景设定图。
- 需要把多张参考资产压缩成一张主要参考图。

Avoid when:

- 用户只要快速抽样。
- 项目还没有角色或场景锚点，角色漂移风险高。
- 需求是广告/短剧项目总览，而不是单段执行。
- 主要卖点是爆炸、粒子、烟雾、混乱战场、强抽象动态或大量随机流体，且镜头顺序无法被清楚画出来。此时优先用设定图/首帧/参考视频 + `../../video-gen-director/references/director-method.md` 的 prompt-heavy 路径，或只把故事板作为角色/场景锚点。

## Required Inputs

- 段落目标：这一段 4-15 秒要让观众看到什么。
- 角色锚点：脸型、五官、发型、服装、配饰、体态、状态。
- 场景锚点：空间结构、光源方向、关键道具、天气、主色调。
- 风格规则：画幅、题材、色彩、材质、镜头速度、颗粒或景深。
- 动作链：起点、推进、转折、反应、细节、收尾。

没有设定图时，可以文生图起步，但仍要有清楚的人物设定、场景设定和段落目标。已有设定图时，优先图生图起步。

## Default Grid Structure

默认九格适合需要完整起承转合的单段视频。使用时可以合并、删除或扩展格子；改写后在提示词里保留“第几镜头是什么、顺序如何读”的说明。

| Cell | Function | Typical Content |
| ---- | -------- | --------------- |
| 1 | 环境交代 | 全景，地点、时间、氛围 |
| 2 | 人物入场 | 中景，角色位置、姿态和当前状态 |
| 3 | 关系建立 | 角色看向目标、靠近目标或发现异常 |
| 4 | 动作启动 | 角色开始执行关键动作 |
| 5 | 转折点 | 强视角、低角度、高角度或情绪爆点 |
| 6 | 反应镜头 | 表情变化、犹豫、惊讶、警觉 |
| 7 | 情绪核心 | 面部特写或眼神特写 |
| 8 | 细节镜头 | 手部、道具、门把手、脚步、影子 |
| 9 | 收尾钩子 | 情绪落点、悬念、动作闭环或下段入口 |

可以按剧情调整。核心要求是同一段里的连续瞬间，而不是九张不同海报；格数和布局服务场景，不反过来限制场景。

## Confirm Before Generation

生成九宫格或自定义分镜母图前，先确认：

```text
分镜母图确认：
- 视频段落：SDxx / 单段
- 时长和画幅：
- 角色锚点：
- 场景锚点：
- 实际镜头数量：
- 读图顺序：从左到右、从上到下 / 自定义说明
- 镜头列表：1 ... N
- 本段结束状态：
- 图片规格：4K 或更高
- 图内文字：无 / 简短编号标签 / 清晰镜头表
- 视频提示词策略：短读图规则；只在复杂特效时补充必要文字
```

用户确认后再生成分镜图。分镜图生成后仍要二次确认图片本身是否符合镜头顺序；确认通过后再写 `video-prompt.txt`。

## Image Prompt Template

```text
基于参考人物图和参考场景图，生成一张 4K 电影分镜参考图，以 3x3 九宫格为默认结构。
画面属于同一个 <时长> 秒视频段落，按从左到右、从上到下形成连续叙事；如果当前场景更适合 <镜头数量> 个关键瞬间，可以合并或调整格子，但读图顺序必须清楚。

角色要求：锁定 <角色名> 的 <脸型/五官/发型/服装/配饰/体态/肤色/比例>。
允许表情和动作随剧情变化，但禁止换脸、换服装、换年龄、换身材。

场景要求：锁定 <场景名> 的 <空间结构/主光方向/关键道具/材质/色调/天气>。
允许镜头景别变化，但禁止切换到无关空间。

默认九格内容（可按场景裁剪或合并）：
1. <环境交代>
2. <人物入场>
3. <关系建立>
4. <动作启动>
5. <转折点>
6. <反应镜头>
7. <情绪核心>
8. <关键细节>
9. <收尾钩子>

画面要求：结构清晰，4K 高分辨率。允许清晰编号、短标签或小型镜头表来说明读图顺序和镜头功能，但文字必须可读、图文一致；禁止乱码、无意义伪字、水印、黑边。
电影感构图，统一光线，统一色彩，统一颗粒，统一景深。
整张图像是同一支短片的连续关键帧，而不是无关海报拼贴。
```

## Seedance Video Prompt Template

```text
参考图1是 @分镜母图，用于决定一段 <时长> 秒 <画幅> 视频的镜头顺序、动作节奏和调度。
参考图2..N 是本段角色卡、场景/背景卡和必要道具/产品卡，用于锁定人物、空间、背景层次和关键道具细节。

读图规则：
- 分镜格子或镜头表决定镜头顺序和动作连续性。
- 角色卡参考图优先锁定人物外观，不让分镜母图里的简化人物覆盖角色卡。
- 场景/背景卡参考图优先锁定空间结构、光线方向和关键道具，不让分镜母图里的简化背景覆盖场景卡。
- 如果实际不是九格，按图内编号或自定义读图顺序执行。

保持角色卡中的人物身份、脸型、服装和体态一致；保持场景/背景卡中的空间、光线方向、主色调和道具一致。
动作只在分镜母图已有内容之间补足，不新增无关角色，不切换无关场景，不改变角色造型，不改变故事结果。
```

## Quality Checklist

- 当前格数是否能讲清同一个 4-15 秒段落。
- 第一格是否建立环境，最后一格是否形成收尾或悬念。
- 角色是否全程同脸、同服装、同比例。
- 场景是否全程同一空间，光源方向不跳。
- 是否有景别变化，而不是九张相似半身图。
- 是否包含至少一个情绪特写和一个关键细节镜头。
- 是否是 4K 或更高规格，文字、编号或镜头表是否清晰可读。
- 视频提示词是否保持简短，只说明读图顺序、运动目标和禁止项，没有逐字复述整个故事板提示词。

## Common Failures

| Symptom | Fix |
| ------- | --- |
| 分镜无时间顺序 | 重写镜头功能，按起点、推进、转折、收尾排列 |
| 角色每格都不同 | 回到 `../../video-gen-assets/references/character-sheet.md` 固定人物锚点 |
| 场景每格跳变 | 回到 `../../video-gen-assets/references/scene-sheet.md` 固定空间 DNA 和光源 |
| 每格景别相同 | 加全景、中景、特写、细节和强视角 |
| 视频乱读分镜 | 在 `video-prompt.txt` 按实际镜头数量明确顺序 |

## Registration

生成九宫格图片后，进入 `../../video-gen-assets/references/seedance-handoff.md`：

```bash
videogen storyboard add <project-id> --storyboard-id <id> --image <grid.png> --prompt <grid-prompt.txt> --video-prompt <video-prompt.txt>
```
