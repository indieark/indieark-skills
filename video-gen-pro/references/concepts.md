# Core Concepts: 剧本 / 资产 / 方法论

> Load when: 需要厘清一次视频任务里“内容、料、组织方式”分别是什么、谁属于谁、谁可以省略。
> Avoid: 已经清楚三者边界，只需要某一类的具体方法；直接读对应文件。
> Pairs with: `../../video-gen-script/references/generation-workflow.md` 把三者推进成流程；`../../video-gen-assets/references/assets.md` 是资产族运行入口；`methods.md` 是方法论运行入口；`../../video-gen-cinematography/references/input.md` 是横切维度输入指南的运行入口。

一次视频任务由三个**互不归属**的一级概念构成。先分清它们，再谈各自的形态。任何上层文件提到“角色卡 / 场景卡 / 故事板 / 剧本”时，归类一律以本文件为准。

## 三个一级概念

| 概念 | 是什么 | 关键性质 | Canonical / 入口 |
| ---- | ------ | -------- | ---------------- |
| 剧本 | 纯文字内容：故事、对白/旁白、镜头文字描述 | 一次性、绑定本片；回答“要讲什么” | `../../video-gen-script/references/generation-workflow.md` 剧本→资产清单→镜头引用；CLI `script` |
| 资产 | 可登记、可**跨剧本复用**的实体 | 有独立生命周期：登记、检索、复用；是“被反复调用的料” | `../../video-gen-assets/references/assets.md`、`../../video-gen-assets/references/`；CLI `asset` |
| 方法论 | 把剧本**组织/呈现**出来的方法 | 可换、可不用；回答“怎么组织” | `methods.md`（方法选择门；导演法/故事板法两种） |

剧本引用资产，方法论组织剧本——但资产不是剧本的附属物，方法论也不是必须项。

**资产从剧本拆，不从镜头反推。** 读剧本就能确定本片有哪些角色、场景、道具和声线，这层与后面怎么分镜无关；镜头只是这些资产的使用者，从剧本资产清单里引用，不定义新资产。**「剧本」还是一个可伸缩的单元**：用户只要一条片时，这条片本身就是剧本，资产只盘它需要的；要系列/多段时，剧本是整个世界，资产盘的是跨段复用的那批。无论大小，资产识别的入口都统一在「剧本」这个单元，所以资产天然挂在剧本上、成为可跨剧本检索复用的单元。

这里要分清**来源**和**生成时机**两件事：来源固定——资产清单一律从剧本拆；生成时机灵活——既可以照清单一次性批量预生成（复用性好、一致性稳），也可以等某个镜头真用到时再懒生成清单里已规划的那个资产（轻量/探索任务省心），两种都是正式做法，按任务规模选。另外，「剧本→资产清单→镜头引用」是**默认心智模型而非强制流水线**：它指导"资产该从哪来"，但轻量、单镜头任务不必显式产出一份独立的"资产清单"文档，凭剧本心里有数直接出资产即可——方法论可换可省的精神同样适用于这条。

## 资产：维度 → 形态

资产先分**维度**，每个维度下的**形态**是一条**从轻到重（L0 → L2）的梯度**：L0 最轻（一次性、单角度、低复用），L2 最重（跨段/系列复用的 master）。按复用强度、镜头/视图角度数和一致性风险选档；不要默认最轻档，也不要无脑上最重档。这条梯度只管一个资产**做**多重；实际**喂**视频模型时按当前镜头需求精选哪几张、喂多细，是另一条轴，见 `../../video-gen-cinematography/references/input/image-input.md` 的 Input Economy。

| 维度 | 状态 | L0 轻 | L1 中 | L2 重（master） | Canonical |
| ---- | ---- | ----- | ----- | --------------- | --------- |
| 人物 | 已实现 | 单图草稿 | 三视图 | 设定板（多视图） | `../../video-gen-assets/references/character-sheet.md` |
| 场景 | 已实现（含背景） | 单一场景概念图 | 多视角联合图 | 正交四视图 | `../../video-gen-assets/references/scene-sheet.md` |
| 道具 | 已实现 | 纯净底单图 | 主图 + 右 1/3 顶/侧拆解 | 加底视图的多视图道具卡 | `../../video-gen-assets/references/prop-sheet.md` |
| 素材 | 已实现（轴=复用范围） | 一次性单件 | 项目套件 | 跨项目系列风格 master | `../../video-gen-assets/references/graphic-sheet.md` |

人物按需选档：L0 单图草稿是**非正式草稿档**，龙套或只出现一次的角色可用，甚至带背景的人图都行；正式角色卡从 L1 三视图起，L1 只取常态表情、无表情变化，无特殊演出需求时已够用；L2 多视图 master 本身缺大范围面部写真、脸易走形，**必须额外捆绑高清头肩胸/面部特写单图**（做几张典型表情、跟 master 走、可跨镜头复用）锚定面部，不可单独交付。人设描述提示词是 L1/L2 的文本底料，不单独算图形态。场景的 L0 单一场景概念图内含快速抽卡（探索写法）和基本场景卡（标准写法）两种写法变体。道具的硬规则是**纯图像零文字**（同角色卡/场景卡）；素材的梯度轴是**复用范围**而非视图角度数，按"一次性 / 项目级 / 跨项目"选档。各维度的选档判据细则见对应 Canonical 文件。

声线不进上面的图像维度表：它是**人物的附属音频子资产**，单一形态（音频样本，无 L0→L2 梯度），登记时用 `--role` 绑定角色名，作为 omni 模式的 `reference_audio` 锁定音色。

当前**可登记、可复用的资产类型共五种**（CLI `ASSET_TYPES = ("character", "scene", "prop", "graphic", "voice")`）：人物、场景、道具、素材四类图像资产，加声线一类音频资产。文档其他位置出现的“产品卡、背景卡、声音 brief”已分别归位：产品并入道具，背景并入场景，声音 brief 落为声线（voice）资产或 prompt 输入。

> 术语预警：素材进 CLI 时英文类型名取 `graphic`，刻意避开既有的 `material` 字段——后者是「任意资产的存储文件 / 来源路径」记录（`material.stored_path`、`material.sha256`），是所有资产共有的存储元数据，不是某个维度。两者不要混。

## 方法论：两种并列方法

方法论是「一段视频如何生成」的具体方法，两者**互不归属、各不相同**。核心分界是**要不要先用资产再生成一层图像中间产物（故事板/分镜图）**：导演法**不**生成这层中间产物，用文字描述为主、资产输入为辅直接控场；故事板法**要**先用资产生成图像母图，再用它主控镜头顺序和运动。注意导演法不是「不用资产」——它同样输入参考图/视频/音频、角色卡、场景卡、道具卡，只是不把这些资产再合成一张可视化的故事板/分镜母图。提示词跟着方法走：每种方法自带自己的**提示词结构**和**素材使用方式**。一次任务只选最适合的一种作为主结构，可吸收另一方法的思路但不堆叠模板。

**并列不是控制力排序。** 两种方法是按**场景适配**分工的并列选项，不构成「导演法→故事板法」这样控制强度递增的光谱或刻度。选择依据是「当前场景/诉求适合哪种组织方式」，不是「哪种控制力更强」——例如粒子、爆炸、抽象动态这类画面画不清的场景，反而走导演法 prompt-heavy，而非升级到故事板法。各方法**内部**对自身控制方式的相对描述（如导演法说自己「靠语言控制、不画死每帧」）只定位该方法自身，不可外推成两方法之间的强弱排序。

| 方法 | 镜头形态 | 提示词结构 | 素材使用 | Canonical |
| ---- | -------- | ---------- | -------- | --------- |
| 导演法 | 单/少镜头控场，时序递进写在画面内容里 | 详细文字描述为主、图片为辅（prompt-heavy）：三段式骨架（【基础设定】资产 + 【氛围与画质】观感一致性 + 【画面内容】分镜镜头），每个资产都认真详细描述 | 角色卡/场景卡作参考图，不依赖图像母图 | `../../video-gen-director/references/director-method.md` |
| 故事板法 | 图像母图主控镜头顺序和运动 | 图像母图为主、短读图 prompt 为辅；资产只需简单描述（图已画详细）；含九宫格/四栏/脚本拼版等版式与多段编排 | 故事板母图作主运动参考图 + 角色卡/场景卡/道具卡细节锚点 | `../../video-gen-storyboard/references/`（`../../video-gen-storyboard/references/storyboard-board.md` 为母文件） |

资产描述详略随方法分情况：先按场景筛掉用不到的资产（无参考素材/无声等可省略，详见 `../../video-gen-director/references/director-method.md`「按场景取舍」），再对**留下的每个资产**定详略——**导演法没有图像母图兜底，本片实际出现的每个资产（角色/场景/道具/声音）都要靠文字认真详细描述**；**故事板法因为故事板/分镜母图已经把画面画得很详细，资产只需简单描述**。

「分镜」一词易混，按形态区分归属：**分镜化文字描述**（导演法【画面内容】里逐镜/连续时序的文字）是**导演法**的写法，纯文字、不画图；**分镜图/故事板母图**（九宫格/四栏/拼版等画出来的格子图）属于**故事板法**。九宫格、四栏、8 镜头、制作设定板、脚本拼版只是故事板法的版式实例，不是独立方法。方法论可换、可省（“不一定使用任何方法”）；轻量任务可以不加载任何方法。

### 两法权衡：代价与适用

两法都用资产，区别只在「要不要先把资产合成一张可视化母图」。这带来不同的代价和适用面，按场景选，不是按「哪种更高级」选：

| 维度 | 导演法 | 故事板法 |
| ---- | ------ | -------- |
| 主控手段 | 文字描述为主、资产输入为辅 | 图像母图为主、短读图 prompt 为辅 |
| 资产用法 | 参考图/视频/音频、角色卡/场景卡/道具卡直接作输入 | 先用这些资产生成故事板/分镜母图，再连同细节卡一起输入 |
| 画面可预见性 | 低：生成前看不到画面，靠语言框定 | 高：母图把镜头顺序和运动提前可视化，间接预览最终帧 |
| 耗时 | 短：不生成中间图像，直接进视频生成 | 长：要多轮生图做母图/分镜图，再进视频生成 |
| 抽卡层数 | 一层（视频模型） | 两层（先抽母图，再抽视频）——母图本身也是抽卡产物 |
| AI 发挥空间 | 大：语言留白，AI 在框架内发挥 | 小：母图锁死镜头顺序和运动，发挥空间收窄 |
| 适用 | 抽象动态（粒子/爆炸/烟雾）等画不清的画面、强素材/参数控场、轻量快速 | 需要严格画面一致性的系列/多段复用、可规划的清晰镜头 |

注意：故事板母图只锁**镜头顺序和运动**，不单独锁人物/背景/道具细节——这些仍靠角色卡/场景卡/道具卡作额外参考图锚定（见 `../../video-gen-storyboard/references/README.md`）。所以故事板法是「大幅收窄不确定性」，不是「100% 所见即所得」。

## 横切维度：输入指南（Input Guide）

剧本/资产/方法论是三根**并列**的轴；输入指南是叠在它们之上、**横切所有方法论**的第四根轴，性质不同，不进上面的三分表。

**方法论是纵，输入指南是横。** 方法论（导演法/故事板法）定义提示词的**骨架结构**和有哪些槽位；输入指南是这些槽位「填什么、怎么填」的**参考知识库**——景别/构图/运镜各选项的语言效果与选用判据、整片风格怎么定、参考图怎么投、时间光影/地点氛围 token。它被两种方法论**共同调用**，不属于任何单一方法：

- 景别/构图/运镜（`../../video-gen-cinematography/references/input/shot-scale.md` 景别含焦点、`../../video-gen-cinematography/references/input/composition.md` 构图含机位、`../../video-gen-cinematography/references/input/camera-move.md` 运镜含词库）：导演法在【画面内容】填，故事板法画母图时每格也要定。
- 风格库（`../../video-gen-cinematography/references/input/style-core.md` 风格核心/画风、`../../video-gen-cinematography/references/input/visual-tone.md` 视觉基调）：两法都要定整片不变的画风与器材质感，设定流的视觉圣经也引用。
- 光影调色（`../../video-gen-cinematography/references/input/lighting-tone.md`）：两法都要给段落定随环境微调的色彩与影调，从风格库拆出独立成文件。
- 图片指南（`../../video-gen-cinematography/references/input/image-input.md`）：两法最后都往视频模型喂参考图，都遵守图内零文字（Reference Purity）和按需投放（Input Economy）。
- 时间氛围 token（`../../video-gen-cinematography/references/input/scene-tokens.md`）：场景图的时间/氛围字段引用。

它不与剧本/资产/方法论并列同类——剧本回答「拍什么」、资产回答「用什么料」、方法论回答「用什么骨架」，输入指南回答「骨架的每个槽怎么填好」。挂在任一单方法下，另一方够不着或要重复，所以独立成横切维度。运行入口 `../../video-gen-cinematography/references/input.md`、二级索引 `../../video-gen-cinematography/references/input/README.md`。

## 资产 vs 方法论产物：复用性判据

二者最容易混的是**故事板母图**：它由方法论产出，又作为参考图提交给模型，行为像资产。判据是复用性：

- **能跨剧本复用 = 资产**（人物卡、场景卡）。
- **绑定本剧本、一次性产出 = 方法论产物**（故事板母图、分镜图）。

故事板母图绑死当前剧本、不跨剧本复用，因此是方法论产物，不登记为资产。

## 现状与迁移方向

三分已按 skill 物理拆分完成：资产族（人物/场景/道具/素材四类图像资产 + 声线音频资产）独立成 video-gen-assets（入口 `../../video-gen-assets/references/assets.md`），方法论两方法各自独立成 skill（导演法在 video-gen-director 的 `../../video-gen-director/references/director-method.md`，用三段式骨架做单/少镜头控场、时序递进写在画面内容里；故事板法在 video-gen-storyboard 的 `../../video-gen-storyboard/references/`）。在三分之上，video-gen-cinematography 承载横切维度**输入指南**（入口 `../../video-gen-cinematography/references/input.md`）（景别填法 `../../video-gen-cinematography/references/input/shot-scale.md`、构图填法 `../../video-gen-cinematography/references/input/composition.md`、运镜填法 `../../video-gen-cinematography/references/input/camera-move.md`、画风/风格核心 `../../video-gen-cinematography/references/input/style-core.md`、视觉基调 `../../video-gen-cinematography/references/input/visual-tone.md`、光影调色 `../../video-gen-cinematography/references/input/lighting-tone.md`、时间氛围 token `../../video-gen-cinematography/references/input/scene-tokens.md`、图片指南 `../../video-gen-cinematography/references/input/image-input.md`）——它不是第四个并列资产/方法，是被所有方法论共用的填法库，物理上独立成目录。五类资产均已落地到 CLI（`ASSET_TYPES`）与对应 Canonical 文件，资产与方法论在目录与措辞上已归位。本文件是三分概念与横切维度的 canonical 源，其他文件只引用、不重述。
