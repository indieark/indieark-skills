# Director Methods 方法选择门

> Load when: 已确定场景，需要在导演法、故事板法之间选择，或判断是否不用方法。
> Avoid: 用户只要轻量一句 prompt、API 参数说明或反馈修正。
> Pairs with: `concepts.md` 定义剧本/资产/方法论三分概念与两种并列方法；`scenes.md` 提供场景信号；`../../video-gen-script/references/generation-workflow.md` 是端到端流程 canonical。

两种**并列、各不相同**的方法论已各自独立为卫星 skill（细则在 `../../video-gen-director/` 与 `../../video-gen-storyboard/`），本文件只做选择门——「一段视频如何生成」的具体方法。核心分界是要不要先用资产生成一层图像中间产物（故事板/分镜图）：导演法不生成这层，用文字描述为主、资产输入为辅直接控场（三段式骨架：基础设定 + 氛围与画质 + 画面内容），故事板法先用资产生成图像母图再用它主控镜头顺序和运动。导演法同样用参考图/视频/音频和角色卡/场景卡/道具卡，只是不把它们再合成可视化母图，且因为没有母图兜底，本片实际出现的每个资产都要靠文字认真详细描述——先按场景筛掉用不到的资产（详见 `../../video-gen-director/references/director-method.md`「按场景取舍」），再把留下的每个资产写详细（故事板法资产只需简单描述，因为图已画详细）。提示词跟着方法走：每种方法自带自己的提示词结构和素材使用方式。`../SKILL.md` 和本文件只负责选择方法，不把所有方法融合成一份大模板。

## 场景先行

方法论不是固定填空模板。先判断用户要的场景，再选择要问什么、保留什么、删掉什么、怎么写 prompt。

场景路由细则见 `scenes.md`。当用户指定产品广告、短剧、角色动作、氛围片、社媒短视频、视频编辑/延长或多参考素材时，先按对应场景收束，不要把所有请求套成同一个“电影感分镜”公式。一次任务只选择最适合的一种方法作为主结构，可以吸收多种方法的判断思路，但避免模板堆叠。

需要人物设定图、场景设定图时，改读 `../../video-gen-assets/references/assets.md` 写图像提示词；需要故事板母图、脚本拼版或多段编排时，走故事板法 `../../video-gen-storyboard/references/README.md`，再回到所选方法写 Seedance 视频 prompt。

## 意图澄清模板

当用户只说“生成一个视频”时，优先问。用户已经给了明确场景时，只补真正影响结果的缺口：

1. 这是广告、故事、产品展示、角色动作，还是氛围片？
2. 用什么画幅和时长？如果不确定，短视频默认 `9:16`、6-8 秒。
3. 要有声还是无声？有声需要音乐、环境音、动作音效、对白，还是参考音频节奏？
4. 有参考图、参考视频、参考音频吗？每个素材要参考什么？

## Method Index

| Method | File | Primary scenes | Use When | Do Not Use When |
|--------|------|----------------|----------|-----------------|
| 导演法 / Director | `../../video-gen-director/references/director-method.md` | Character Action / Atmosphere / Video Edit / Reference-Driven / Product / Short Drama / Social Short | 以**语言控制**为主、不预先画死画面：靠讲究的三段式 prompt 与资产灵活运用做单/少镜头控场，时序递进写在【画面内容】里；素材/声音/限制多，或没有故事板母图需要 prompt-heavy 详细描述 | 以图像母图为主控镜头顺序和运动（改用故事板法） |
| 故事板法 / Storyboard | `../../video-gen-storyboard/references/README.md` | 复杂项目 / 系列内容 / 需要一致性的可规划镜头 | 以图像母图主控镜头顺序和运动；需要九宫格/四栏/脚本拼版等版式或多段连续性 | 粒子、爆炸、烟雾、混乱群体或强抽象动态为主时，优先 prompt-heavy 或混合路径 |

故事板法的母图理念、版式实例（九宫格、四栏、脚本拼版）和智能编排见 `../../video-gen-storyboard/references/README.md`。「分镜」分两种归属：导演法【画面内容】里的分镜化文字（含连续时序递进）是纯文字写法，分镜图/九宫格等画出来的格子图属故事板法。

## Selection Rule

先读 `scenes.md` 判断场景，再选择方法：

```text
场景明确 -> 选择一个最轻的方法 -> 只填会影响生成结果的字段 -> 写入 Seedance prompt
```

不要把多个方法完整叠加到一个 prompt。可以吸收多个方法的思路，但输出必须服务用户当前场景。
