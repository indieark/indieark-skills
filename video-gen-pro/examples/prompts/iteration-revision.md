# Iteration Revision Example

演示如何把用户对第一次结果的反馈映射到 1-3 处定向修改，而不是重写整段 prompt。完整诊断映射见 `references/iteration.md`。

## Round 1 (initial)

### User Intent

8 秒电影感角色奔跑短片，用参考图保持主角外观，用参考视频参考动作节奏，用参考音频参考鼓点，9:16 有声。

### Director Prompt (R1)

```text
成片目标：8 秒霓虹雨夜街道角色奔跑短片，紧张感。
参考关系：参考图1用于主角外观；参考视频1用于跑步动作；参考音频1用于鼓点。
画面与风格：雨夜城市，电影感写实。
镜头设计：低角度跟拍，缓慢推进。
动作节奏：主角调整呼吸 -> 加速奔跑 -> 冲向远处光源。
声音设计：参考音频1的鼓点节奏 + 雨声 + 低频氛围。
限制：保持主角身份一致。
```

## User Feedback

> 主角看起来不像参考图里的人，动作也太慢了，节奏没跟上鼓点。

## Diagnosis

按 `references/iteration.md` 反馈映射：

| 症状     | 可能原因                                       | 选定修复                                        |
| -------- | ---------------------------------------------- | ----------------------------------------------- |
| 不像原图 | `reference_image` 用途说得太浅，被风格漂移盖过 | 强化参考图角色描述，列出"保持/不改变"的具体特征 |
| 动作太慢 | prompt 把"奔跑"写成了静态形容                  | 拆出明显的节奏阶段，引入鼓点对齐动词            |
| 没跟鼓点 | 没有显式把画面动作绑定到参考音频节拍           | 用"每个重拍"做画面动作锚点                      |

只改 3 处，不重写其余部分。

## Director Prompt (R2)

```text
成片目标：8 秒霓虹雨夜街道角色奔跑短片，紧张感，主角身份与参考图严格一致。

参考关系：
- 参考图1用于主角面部、发型、外套主色和体型，不允许漂移到其他风格或其他人物。
- 参考视频1只用于跑步动作节奏与侧后方跟拍方式，不复制原视频背景。
- 参考音频1用于鼓点节奏与紧张情绪。

画面与风格：雨夜城市，电影感写实，浅景深，地面积水反射霓虹。

镜头设计：低角度侧后方跟拍，开场中景；2 秒后镜头推近，5 秒后随主角加速一同提速。

动作节奏：每一个鼓点重拍触发一次画面推进——
- 0-2 秒：主角调整呼吸，第 1 个重拍开始迈步。
- 2-5 秒：脚步踩过积水，每个重拍水花更高。
- 5-8 秒：随鼓点密度加快，主角加速冲向远处白色光源。

声音设计：使用参考音频1的鼓点节奏，加入低频氛围与雨声，不生成对白。

限制：保持主角身份一致；不加入无关文字、字幕、logo；不改变服装主色。
```

## CLI Shape

```bash
videogen omni \
  --prompt "<R2 director prompt>" \
  --reference-image character.png \
  --reference-video motion.mp4 \
  --reference-audio beat.mp3 \
  --generate-audio true \
  --duration 8 \
  --ratio 9:16 \
  --wait
```

## Self Check

- 是否每条反馈都有 1 处对应的 prompt 修改，且总修改不超过 3 处。
- 是否没有删掉用户已经认可的部分（场景、风格、声音方向）。
- 是否没有引入新的互斥参数（例如同时加 first_frame）。
- 是否在 prompt 中显式标注了"保持/不允许漂移"，对抗参考图弱化。
