# Director Method 导演法

> Load when: 已完成交互/场景判断，选择以详细文字描述为主、图片为辅的 prompt-heavy 路线，靠语言控住画面、镜头、声音。
> Avoid: 以图像母图为主控镜头顺序和运动（改用 `../../video-gen-storyboard/references/storyboard-board.md`）；仍不清楚用户目标时先回到 `../../video-gen-pro/references/interaction.md` / `../../video-gen-pro/references/router.md`。
> Pairs with: `../../video-gen-pro/references/methods.md` 决定选哪种方法；`../../video-gen-pro/references/scenes.md` 决定保留哪些字段；`../../video-gen-pro/references/sound-design.md` 负责有声、无声、音效、对白和参考音频规则；`../../video-gen-assets/references/scene-sheet.md` 生成或登记场景卡。

导演法是两种方法论之一：把一次生成当作一次短镜头拍摄前的控场，用**详细文字描述为主、图片为辅**把基础设定、氛围画质、画面镜头、声音和限制压成一份**三段式 prompt**（见下「Prompt 结构」）。它是没有故事板母图时的主路线（prompt-heavy）。故事板法是另一种方法，用图像母图主控镜头顺序和运动，有自己的提示词结构和素材使用方式；一次任务只选最适合的一种作为主结构。

**导演法的控制方式是相对的。** 这里描述的是导演法**自身**怎么控场，不是给两种并列方法排控制力强弱（两者并列、互不归属、按场景适配选择，见 `../../video-gen-pro/references/concepts.md`）。导演法用**语言控制**，不把每一帧画面提前画死，靠最讲究的**提示词方法**和对**各种资产（参考图/视频/音频、角色卡、场景卡、道具卡）的灵活运用**，把镜头目标、主体、动作、声音、限制交代清楚，让 AI 在清晰框架内发挥；与之相对，故事板法改用**图像母图**把画面顺序和运动预先锁定，是另一种并列的控场方式而非「更强一档」。所以本文下面说的「避免模型随意改场景/主体/动作」是**语言层面**要达成的目标，不是要逐帧锁死——导演法靠语言把控场交代清楚，落在「放手发挥」和「图像锁死」之间，而非强控每一帧。

## When to Use

Use when:

- 用户指定了具体场景、素材、声音或强限制。
- 需要避免模型随意改场景、改主体、改动作。
- 需要把参考图、视频、音频的用途讲清楚。
- 需要一个连续镜头或少量镜头，而不是复杂长剧情。
- 没有故事板母图，必须靠详细文字 prompt 控住画面、镜头、声音和文字。

Avoid when:

- 用户只是在探索创意方向。
- 用户以图像母图为主控镜头顺序和运动（改用 `../../video-gen-storyboard/references/storyboard-board.md`）。
- 用户请求极轻量，只缺一句 prompt。

## 写 Prompt 前置条件

复杂生成任务写最终视频 prompt 前，必须已经完成 `../../video-gen-script/references/generation-workflow.md` 的导演流程（该文件是 canonical）：自然语言转剧本、剧本转资产清单、剧本转镜头、镜头引用资产。资产从剧本拆、镜头只引用，不从镜头反推。没有故事板时，也不能省略这个步骤；只是把镜头清单和资产引用直接转为下方三段式骨架视频 prompt。

如果任意非抽象镜头没有 `scene_ref`，先回到 `../../video-gen-assets/references/assets.md` / `../../video-gen-assets/references/scene-sheet.md` 生成或登记场景卡。不要用一句“在某个场景中”替代场景资产，也不要把缺失的场景、道具和空间关系全塞进最终 prompt。

## Prompt 结构：三段式骨架

导演法（无故事板）的最终视频 prompt 用三段式骨架组织，分工固定、自上而下收束：**【基础设定】**写资产、**【氛围与画质】**写全片观感一致性、**【画面内容】**才写分镜与镜头。三段各管一层，不要把镜头细节塞进基础设定，也不要把资产描述散进画面内容。

**三段之外零追加。** 最终 prompt 有且只有这三段：不在三段前后追加项目名、段落编号（如「这是某片的第 SD01/N 段」）、承接方式、「上一段/本段结束状态」、单列「禁止」清单或任何面向执行者的说明性文字。这些是连续性台账和生成记录的内容（见 `../../video-gen-script/references/multi-segment-continuity.md` 的 Segment Ledger），不发给模型。约束要折进对应段位：资产不可改变的特征写进【基础设定】、本段起始画面与结尾落点及不出现的元素写进【画面内容】的「画面内容」字段、文字/水印禁止项写进「画面文字」字段。

每段都按场景取用必要字段，不要求机械全填——但导演法没有故事板母图兜底，凡是写到的资产和画面都要**认真详细描述**，靠语言把模型需要知道的交代清楚（这是导演法区别于故事板法的关键：故事板法因为图已经画得很详细，资产只需简单描述）。

### 【基础设定】

资产层：把本片用到的角色、道具、场景、声音逐个索引并**认真详细描述**。核心原则是**以文字为主、图片只作索引锚点，不盲信依赖图片**——`@xxx图片` 只是指向哪张参考图，真正约束模型的是后面的文字。每个资产（含声音、声线等所有索引部分）都要靠文字写清，不能只丢一张图。

```text
角色：「@角色图片」+ 详细文字（体型、风格、服装、不可改变的特征、面部/表情规则）。
道具：「@道具图片」+ 详细文字（外形、尺寸、材质、状态变化，如启动/充能/爆炸过程）。
场景：「@场景图片」+ 详细文字（年代风格、时间、天气、空间状态、关键陈设与氛围细节）。
声音：配乐 / 氛围音 / 音效 / 对白 / 同期声 / 参考音频用途；无声或仅同期声也在此写明。
```

声音入【基础设定】，因为声音、声线是可后续填充的资产；具体写法和有声/无声/音效/对白/参考音频规则仍以 `../../video-gen-pro/references/sound-design.md` 为 canonical，不在此重复。

### 【氛围与画质】

全片观感一致性约束：在同一影视中约束观感一致，跨多段/多次生成时**整块复用**（见下「跨段复用」）。四个固定子字段，按变化策略区别对待：

```text
风格核心：整片基本完全不变。题材、质感方向、写实/CG 取向、要杜绝的观感。
视觉基调：整片基本完全不变。摄影机/镜头/画幅取向、胶片或数字质感。
色彩与影调：只在环境变化时，描述部分有轻微变化。色调、对比、光照、滤镜、颗粒。
氛围基调：跟据剧本需要随时改变。本段/本片要凸显的情绪与张力。
```

风格核心（整片不变的画风/题材流派 + 把抽象词翻译成可见元素的写法）见输入指南 `../../video-gen-cinematography/references/input/style-core.md`、视觉基调（整片不变的器材/画幅/胶片质感）见 `../../video-gen-cinematography/references/input/visual-tone.md`；随段落微调的色彩与影调模板池单独拆在 `../../video-gen-cinematography/references/input/lighting-tone.md`——这里只列槽位，填什么风格走那几处。

### 【画面内容】

才描述分镜与镜头。子字段：

```text
分镜：单镜头一镜到底 / 镜头 A、B / 连续镜头。
景别：远景 / 全景 / 中景 / 近景 / 特写；浅景深、长焦压缩等。
构图：前景/背景的主体分布、画面位置、占比、透视关系、视觉动线。
运镜手法：固定机位 / 跟随 / 推拉 / 平移 / 环绕 / 手持，以及速度。
画面内容：主体、动作与**连续时序递进**——把 5-15 秒里的动作按时间顺序写成一段连贯叙述（如「随即…」「片刻后…」「爆炸后 0.5 秒…」），让动作分阶段推进，而不是单独分拍。
画面文字：text-track（文字/字幕轨道）——画面上可见的标题文字、字幕、UI 文字或品牌字样；没有则写明不出现文字，且不新增未指定文字/水印/logo。这是画面里能看见的文字，不是声音。
```

景别（含焦点/景深）的填法见输入指南 `../../video-gen-cinematography/references/input/shot-scale.md`、构图（含机位角度）见 `../../video-gen-cinematography/references/input/composition.md`、运镜手法（含五类镜头词库速查）见 `../../video-gen-cinematography/references/input/camera-move.md`——各选项的画面语言效果、什么剧情该选哪个、怎么写进 prompt 都在那几处，这里只列槽位。

时序递进直接写在「画面内容」里：用时间词把一个连续镜头内的动作变化、卖点展开、转折或记忆点串成一段，不另起分拍小节。需要起承转合、单镜头放不下时，先回到 `../../video-gen-pro/references/scenes.md` 判断是否该走多段拆分（`../../video-gen-script/references/multi-segment-continuity.md`），而不是在一次 prompt 里硬塞过载。

声音相关的两条轨道不在这里：audio-track（音乐/环境音/音效）和 speech-track（说话人声/对白/旁白/口播）属于声音资产，落在【基础设定】的「声音」项，规则以 `../../video-gen-pro/references/sound-design.md` 为准。本段只管画面里能看见的 text-track。

### 跨段复用

【氛围与画质】是跨段复用区块：同一影视的多段/多轮生成里，**风格核心和视觉基调逐字保持不变**（可直接复制粘贴），只有**色彩与影调随环境变化做局部微调**、**氛围基调随剧本需要改写**。【基础设定】里同一资产的文字描述也应跨段保持一致，避免角色/场景在不同段之间走形。多段连续性的台账与拆分规则见 `../../video-gen-script/references/multi-segment-continuity.md`。

多段生成时段落承接同样只在三段内表达：本段起始画面（即上一段 `end_state` 的可见状态）写在【画面内容】「画面内容」字段的开头，本段落点写在该字段结尾；段落编号、承接方式、`end_state`、`carry_over` 留在连续性台账，不进 prompt。那份模板里的「这是第 X 段 / 上一段结束状态 / 禁止」小节只适用于故事板法的 `video-prompt.txt`，导演法不要套用。

## Prompt 详略策略

<!-- contract: prompt-detail-routing storyboard-concise-prompt no-storyboard-detailed-per-shot-prompt audio-track speech-track text-track -->

提示词详细程度取决于是否使用故事板母图。

- **使用故事板时**：改走故事板法，遵守“一图控制 + 文字解说图片”的 storyboard-concise-prompt 策略，`video-prompt.txt` 只写参考图编号用途、读图顺序、运动目标、短声音 brief 和禁止项；完整规则见 `../../video-gen-storyboard/references/storyboard-board.md` 和 `../../video-gen-assets/references/seedance-handoff.md`。
- **不使用故事板时**：走导演法本路线，按 no-storyboard-detailed-per-shot-prompt，视频 prompt 必须更详细，用上面的三段式骨架逐项写清楚：【基础设定】的资产以及 audio-track（音频轨道）、speech-track（说话人声）这两条声音轨道；【氛围与画质】的观感一致性；【画面内容】的画面、镜头与 text-track（文字/字幕轨道，画面里可见的文字）。三条轨道按归属分别落到对应段，不混在一段写。

### 不使用故事板时

没有故事板母图时，视频 prompt 必须更详细。先从已确认镜头清单转换，不要直接从自然语言创意跳写；用三段式骨架把模型需要的资产、氛围画质、画面、镜头、声音、人声和文字都写清楚，三条轨道按归属分别落位、不漏轨：音频轨道（音乐/环境音/音效）和说话人声（对白/旁白/口播）落在【基础设定】的声音项，文字轨道（标题文字/字幕，画面里可见的文字）落在【画面内容】的「画面文字」字段。完整字段见上「Prompt 结构：三段式骨架」，下面「Example」给出端到端示例。

控制负载：如果一次画面内容的总时长超过 Seedance 单次 4-15 秒限制，先回到 `../../video-gen-script/references/multi-segment-continuity.md` 拆分生成轮次；不要把整部片的所有镜头硬塞进一次 payload。

## 按场景取舍

三段式骨架不是固定填空表。按场景只保留会影响生成结果的字段，不机械全填：

- 无参考素材时，【基础设定】资产只写文字、不强写 `@图片` 索引。
- 无声视频时，【基础设定】声音项明确写“无声 / 仅同期声”，不展开音乐音效。
- 用户要单一安静动作时，【画面内容】只写一个连续动作，不强拆时序递进。
- 用户指定了特定场景/风格时，先保留其边界，不替换成通用广告/电影模板。
- 氛围片省略强卖点；社媒钩子片首 1-2 秒就要在【画面内容】写出钩子；编辑/延长片【基础设定】重点写保留什么、改变什么、延续到哪里。

## 镜头词库（附录）

建立/角色/运动/产品/情绪五类常用镜头词的速查表归入 `../../video-gen-cinematography/references/input/camera-move.md` 末尾；景别（含焦点）各选项的画面语言效果与选用判据见 `../../video-gen-cinematography/references/input/shot-scale.md`、构图（含机位角度）见 `../../video-gen-cinematography/references/input/composition.md`。【画面内容】的景别/构图/运镜子字段从那几处取词、按那里的判据选。

## 声音设计

有声视频先读 `../../video-gen-pro/references/sound-design.md`，再把 audio brief 压进最终 prompt。不要只写“生成声音”，要说明声音承担什么功能：音乐、音效、对白、字幕/标题文字、参考音频用途和无声策略均以 `../../video-gen-pro/references/sound-design.md` 为准。

## 全能参考写法

参考素材在 prompt 中显式编号（第 1 个 `reference_image` 写 `参考图1`、第 1 个 `reference_video` 写 `参考视频1`…），逐张说明用途，比笼统写“参考所有素材”稳定。完整编号规则与示例，连同参考图按需投放（Input Economy）和图内零文字（Reference Purity），已归入输入指南 `../../video-gen-cinematography/references/input/image-input.md`。

## Example

端到端三段式示例（无故事板、单镜头一镜到底、强素材控场）。用户要一个原子朋克末日逃亡镜头：机器人骑鸵鸟在丧尸尸潮前狂奔，途中掏雷管炸弹反手扔向身后引爆。

```text
【基础设定】
机器人清道夫：「@清道夫图片」，身形修长的人形机器人，1960年代原子朋克风格设计。拥有自主意识。面部 LED 显示屏代替五官，显示像素风格的表情（表情无动态效果，保持静帧显示，表情切换带有科幻感音效）。身穿美式西部牛仔复古风格的生锈色牛仔帽、黑色哑光高腰设计皮衣夹克、黑色哑光皮手套、牛仔腰带和枪套。
非洲鸵鸟：「@鸵鸟图片」，成年非洲公鸵鸟，样貌看起来呆傻，双眼歪斜不对称，歪吐着舌头。
雷管炸弹：「@雷管图片」，圆柱体雷管炸弹，长15cm，直径5cm，顶部有按钮开关，按下按钮后启动，内部雷铜线材开始充能发热，散发出橙红色高光，充能到达峰值后爆炸，爆炸威力巨大。
丧尸：「@丧尸图片」，多只腐烂惊悚的丧尸，头发凌乱，皮肤溃烂，肤色灰白暗沉，布满破损伤口与污痕，眼球浑浊发白、瞳孔无光，露出尖锐泛黄的腐烂獠牙，表情狰狞；身着破乱不堪的海滩度假风和日常休闲装，皮肤与衣物沾满血液与腐败肌理。
场景：「@场景图片」，1960年代美式复古原子朋克城市街道，刚刚爆发丧尸危机，大战过后历过一场战斗的死寂。盛夏烈日当空，复古未来主义、复古乌托邦，冷战太空风流线型建筑、混乱场景，破损报废的复古老爷车、散落的报纸、地面有血迹，60年代临街商铺，圆润穹顶科幻楼宇。场景四处均有丧尸的尸体。
声音：不需要配乐，不要氛围音。仅保留同期声。

【氛围与画质】
风格核心：原子朋克、丧尸危机、末日爱情、电影级质感、超写实、极致逼真、Photorealism-真人实景拍摄、杜绝游戏 CG 感、杜绝动作迟钝僵硬。
视觉基调：变形宽荧幕电影质感。使用 IMAX 胶片摄影机，搭配 Panavision C 系列镜头（添加动态模糊）拍摄。
色彩与影调：60年代复古科幻原子朋克美学，复古暖橙 + 冷盐蓝高对比色调，胶片颗粒质感，低饱和度复古胶片滤镜，采用自然日光照明，强直射日光，自然日光光晕；画面明暗对比强烈，高光不过曝，暗部保留完整细节，明暗过渡自然，营造自然立体的光影氛围，带轻微复古胶片柔光质感。
氛围基调：凸显末日逃亡、惊险追逐的刺激感、危机感、紧迫感。

【画面内容】
分镜：单镜头一镜到底。
景别：远景、浅景深。长焦压缩空间。
构图：前景为鸵鸟脖子和机器人腰部以上位置，处于画面中心线右侧，占据画面三分之一面积；背景为城市街道以及远处追来的丧尸尸潮，处于画面中心线左侧。拉伸透视，人物主体与远方延伸的街道形成动态对角。
运镜手法：固定机位，跟随拍摄主体。
画面内容：机器人面部 LED 屏幕保持蓝色冷酷表情，骑乘着鸵鸟面对镜头方向，位于街道中央，沿着笔直的公路狂奔，鸵鸟高速奔跑，翅膀微张，双脚交替踩踏路面，身体带有颠簸起伏，高速移动带有动态模糊感。在其身后的远处，有一片尾随着他们的丧尸尸潮，丧尸们面对镜头方向追杀机器人与鸵鸟，嘶吼声此起彼伏，移动速率前后不一，街道上散落着触目惊心的尸体和血迹。机器人左手握住鸵鸟脖子，右手从身后掏出一根雷管炸弹，单手握持，右手大拇指用力按下顶部启动按钮，炸弹开始充能并发光。随即往身后高高抛出，右手顺势按压住自己帽子，轻微牵肩低头等待爆炸。片刻后炸弹从画面上方入画，精准落在尸潮之中并爆炸，威力惊人，火光绚丽而巨大，丧尸们被炸飞，瞬间产生高能冲击波向四周扩散，卷起地面的灰尘。爆炸后 0.5 秒，强劲的冲击波将鸵鸟与机器人瞬间推出画面，伴随迎面而来的强大推力，画面产生剧烈抖动与失焦。
画面文字：全片不出现任何标题文字、字幕、UI 文字或品牌字样，不添加水印与 logo。
```

注意这条示例的三段分工：资产（机器人/鸵鸟/雷管/丧尸/场景/声音）全部在【基础设定】里靠文字写清，`@图片` 只作索引；全片观感（原子朋克写实质感、IMAX 变形宽荧幕、复古暖橙冷盐蓝、末日紧迫感）在【氛围与画质】里约束；时序递进（按按钮→充能→反手抛出→入画爆炸→冲击波推出画面）写在【画面内容】的「画面内容」字段里连成一段，不另起分拍。

也注意这条示例的**参考图按需投放**（见 `../../video-gen-cinematography/references/input/image-input.md` 的 Input Economy）：五个 `@图片` 不是一个规格。`@清道夫图片`（机器人主角，全程出镜、要严格一致）走 L2 三视图 master，它的 LED 像素表情按镜头需要再配对应的表情单图；而 `@鸵鸟图片`、`@雷管图片`、`@丧尸图片` 都是本镜一次性出镜、不跨段复用，简单单图（L0）就够；`@场景图片` 这条街本镜不复用，也只要一张场景单图，不必上多视角联合图或正交四视图。同一个镜头里，主体角色拉满、配角和场景按需从简——细节投在该投的资产上，不是一律拉满堆成过载。

## Self Check

- 是否尊重用户指定场景，而不是强行套公式。
- 【画面内容】的时序递进是否落在单次时长内（一镜到底通常 5-15 秒内一个连续动作链或一个清晰转折），过载就回到多段拆分。
- 资产、镜头、声音是否互相服务。
- 是否写清参考素材用途。
- 是否写清输出规格、声音、说话人声、字幕/标题文字和不需要的输入。
- 是否把风格词堆太多。
- 是否要求模型同时做互斥任务。
- 是否适合当前 API 模式。
- 三段分工是否清楚：资产在【基础设定】、观感在【氛围与画质】、镜头在【画面内容】，没有串层。
- 最终 prompt 是否有且只有三段：没有段落编号、承接说明、结束状态、单列禁止清单等三段之外的小节或说明性文字。
- 是否没有把三段式骨架当成必须完整填满的表格。
- 是否没有和故事板法重复堆叠。
