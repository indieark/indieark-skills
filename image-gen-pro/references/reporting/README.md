# Reporting Subindex

This directory supports `../reporting.md`. Keep `../reporting.md` as the runtime entry and SSOT index; keep reusable detail here.

## Files

| File | Purpose |
| --- | --- |
| `common.md` | Shared labels, tone, preview rendering, prompt/method/verification rules |
| `templates.md` | Exact user-facing templates for each output family |

## Maintenance Rules

- Do not copy whole templates into `SKILL.md`, `README.md`, or other references.
- When adding a new output family, add it to `../reporting.md` and then add the exact template in `templates.md`.
- Keep each file focused; this folder exists so the reporting entry stays readable and layered.
